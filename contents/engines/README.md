# PhysPlay 엔진 기반 콘텐츠 구조

기존 교과서식 커리큘럼 구조(4트랙 × 6모듈 = 24모듈)를 엔진 기반 콘텐츠 구조(6엔진 × 3레이어)로 재배치한 결과물이다.

## 구조

```
contents/engines/
├── projectile/           ← 투사체 엔진 (Phase 1)
│   ├── concepts.md       핵심 물리 개념
│   ├── challenges.md     변수 조합 챌린지 시나리오
│   └── discover.md       Discover 단계 설명 텍스트 소재
├── collision-energy/     ← 충돌/에너지 엔진 (Phase 1)
│   ├── concepts.md
│   ├── challenges.md
│   └── discover.md
├── wave/                 ← 파동 엔진 (Phase 1)
│   ├── concepts.md
│   ├── challenges.md
│   └── discover.md
├── molecular/            ← 분자 엔진 (Phase 3)
├── orbit/                ← 궤도 엔진 (Phase 4)
├── quantum/              ← 양자 엔진 (Phase 5)
├── _future/              ← 현재 엔진에 매핑 안 되는 미래 콘텐츠
│   ├── electromagnetic/
│   ├── bio-cell/
│   └── cosmology/
└── README.md             ← 이 파일
```

## 각 엔진 폴더의 3개 파일 역할

| 파일 | 역할 | 내용 |
|------|------|------|
| `concepts.md` | 이 엔진이 다루는 물리/화학 개념 전체 목록 | 기존 모듈의 "핵심 개념" 섹션에서 추출 |
| `challenges.md` | 변수 조합 챌린지 시나리오 풀 | 기존 모듈의 "3D 콘텐츠" 아이디어를 변수 조합 형태로 재구성 |
| `discover.md` | Discover 단계 설명 텍스트 소재 | 개념별 직관적 설명, 비유, 심화 설명의 원재료 |

## 출처 매핑 테이블

### 범례

- ✅ 직접 매핑 — 모듈 내용이 해당 엔진에 1:1로 대응
- 🔀 분할 매핑 — 모듈 내용이 여러 엔진에 나뉘어 배치
- ⏳ 미래 엔진 — 현재 6개 엔진에 매핑 안 됨, `_future/`로 보관
- 🔗 크로스 참조 — 다른 엔진의 보조 콘텐츠로 활용

## 크로스 참조 맵

| 개념 | 주 엔진 (정의) | 참조 엔진 (사용) | 연결 맥락 |
|------|---------------|-----------------|----------|
| 적색편이 | 파동 (도플러 효과) | 궤도 (은하 후퇴 속도) | "파동에서 배운 도플러가 우주 팽창의 증거가 된다" |
| 전자기 스펙트럼 | 파동 (파장/진동수) | 양자 (광전효과, 흑체복사) | "파동의 에너지가 양자화되어 있다" |
| 에너지 양자화 | 양자 (플랑크) | 분자 (전자 에너지 준위) | "원자의 전자가 불연속 에너지 준위에 있는 이유" |
| 중력 | 투사체 (표면 중력) | 궤도 (만유인력) | "지구에서의 g=9.8은 만유인력의 특수한 경우" |
| 파동 간섭 | 파동 (물결, 소리) | 양자 (이중슬릿) | "물결 간섭과 같은 원리가 양자 세계에서도 작동" |
| 확률구름/오비탈 | 양자 (파동함수) | 분자 (전자 배치, 결합) | "오비탈의 모양이 화학결합의 방향을 결정" |

**Tagline:** 브라우저에서 과학을 만지고 체험하는 3D/XR 교육 플랫폼

## 1. Problem

### What problem are we solving?

과학 교육에서 가장 중요한 개념들(분자 구조, 양자 현상, 우주 스케일)은 본질적으로 3차원이지만, 현재 교육은 2D 교과서와 영상에 의존한다. 학생들은 수식을 먼저 접하고 직관을 나중에 형성하는 역순 학습을 강요받으며, 이는 과학에 대한 흥미 상실과 높은 이탈률로 이어진다. 기존 시뮬레이션 도구들은 설치형이거나, 단일 주제에 한정되거나, 인터랙션이 부족하다.

### Who has this problem?

**Primary:** 과학 개념을 직관적으로 이해하고 싶은 중고등학생 및 대학 교양 수강생. 특히 수학에 약해서 수식 기반 학습에 벽을 느끼는 학생.

**Secondary:** 과학 흥미를 가진 성인 학습자, 테크/양자컴퓨팅 관심층. 시각적 교보재가 필요한 교사.

### How do they solve it today?

- YouTube 영상 시청 (수동적, 인터랙션 없음)
- PhET Simulations (2D 위주, 설치 필요한 경우 있음, 콘텐츠 분산)
- Khan Academy (영상+문제 풀이, 체험형이 아님)
- 대학 실험실 (접근성 제한, 비용 높음)
- 교과서의 정적 다이어그램 (3D 개념을 2D로 이해해야 하는 근본적 한계)

### Why now?

- **브라우저 3D 기술 성숙:** 브라우저에서 고품질 3D 렌더링과 실시간 물리 시뮬레이션이 가능해짐
- **웹 XR 표준화:** 별도 앱 없이 브라우저에서 XR 체험 가능
- **XR 기기 보급 확대:** Meta Quest, Apple Vision Pro 등 소비자 기기 확산
- **AI 교육 관심 증가:** 양자컴퓨팅, 생체분자 등 테크 관심층 급증
- **오픈 과학 데이터:** PDB, PubChem, NASA API 등 콘텐츠 기반 데이터 풍부

| Hypothesis | Risk Type | Validation Plan |
|-----------|-----------|-----------------|
| 3D 물리 시뮬레이션이 교과서/2D 다이어그램보다 개념 이해도를 높인다 | Value | 사용자 테스트에서 사전/사후 이해도 비교 |
| 브라우저에서 인터랙티브 물리 시뮬레이션을 60fps로 구동할 수 있다 | Feasibility | 물체 던지기 + 롤러코스터 프로토타입 eng spike (2주) |
| 3D/XR 동일 코드베이스로 점진적 몰입 강화 구조를 유지할 수 있다 | Feasibility | XR 모드 전환 프로토타입 검증 |
| Phase 1부터 B2B(학교) 파일럿으로 유료 모델을 검증할 수 있다 | Viability | 교사 파일럿 10곳 확보 후 전환율 측정 |
| 수식 없는 접근이 교육 전문가에게도 받아들여진다 | Usability | 과학 교사 5명 인터뷰 + 피드백 수집 |

## 3. Proposed Direction

### High-level approach

"보이지 않는 것을 체험하게 만든다." 브라우저만 열면 바로 3D 과학 시뮬레이션에 접근할 수 있고, XR 기기가 있으면 몰입이 자동으로 강화되는 **3D First, XR Enhanced** 구조를 만든다. 수식을 먼저 제시하는 대신, 인터랙션을 통해 직관을 형성하고 나중에 수식을 연결하는 **역방향 학습 설계**를 채택한다.

콘텐츠는 4개 트랙(고전물리, 화학, 우주과학, 양자역학)으로 구성하며, 가장 범용적이고 게이미피케이션 잠재력이 큰 고전물리 트랙(운동과 힘, 에너지와 일, 파동)을 첫 번째로 출시한 뒤 순차적으로 나머지 트랙을 확장한다.

### What differentiates this?

- **3D-native 설계:** 기존 교육 도구가 2D에 3D를 얹는 것과 달리, 처음부터 3D로 설계
- **Progressive XR:** 동일 콘텐츠가 데스크톱 → 모바일 → XR로 점진적 몰입 강화. 별도 앱 불필요
- **인터랙션 중심:** 보는 교육이 아니라 만지는 교육. 슬라이더, 드래그, 토글로 변수를 직접 조작
- **통합 커리큘럼:** 단일 시뮬레이션이 아닌, 4개 과학 분야를 관통하는 체계적 학습 경로

### What this is NOT (Non-Goals)

- 기존 LMS(Learning Management System)를 대체하는 것이 아니다
- 학점이나 자격증을 부여하는 공인 교육과정이 아니다
- 전문 연구자를 위한 고정밀 시뮬레이션 도구가 아니다
- 네이티브 앱 (모든 것은 웹 기반)
- Phase 1에서 4개 트랙을 전부 구축하는 것이 아니다

## 5. Audience & Market Context

### Target audience

**Persona: "수학이 싫은 민준" (16세, 고1)**
과학은 재밌을 것 같은데 수식이 나오면 흥미를 잃는다. YouTube에서 양자역학 영상을 봤지만 "결국 수식 없이는 이해 못 한다"는 말에 좌절. 브라우저에서 이중슬릿 실험을 직접 조작해보고 "관측하면 패턴이 바뀐다"를 체감한다면 양자역학에 대한 진입 장벽이 무너진다.

**Persona: "커리어 전환 중인 지영" (29세, 개발자)**
양자컴퓨팅에 관심이 생겼지만 물리학 배경이 없다. 블로흐 구를 직접 회전시키며 큐빗 상태를 조작해볼 수 있다면 추상적 개념이 구체화된다.

### Competitive landscape

| Competitor | Strength | Weakness | Our Angle |
|-----------|----------|----------|-----------|
| PhET (U of Colorado) | 방대한 시뮬레이션, 교육 검증 | 대부분 2D, XR 없음, 산발적 | 3D-native + XR + 통합 커리큘럼 |
| Labster | VR 랩 시뮬레이션, B2B | 고가($50k+/학교), 설치형 | 무료 웹 접근, 설치 불필요 |
| Brilliant.org | 인터랙티브 문제풀이 | 3D 시각화 제한적 | 3D 체험 우선, 수식 나중 |
| YouTube (3Blue1Brown 등) | 뛰어난 시각화, 무료 | 수동적 시청, 인터랙션 없음 | 직접 조작하는 체험 |

### Pricing signal

- **Phase 1 (B2C + B2B):** 기본 무료, 교사/학교 대상 프리미엄 라이선스 (교사 대시보드, 학습 진도 추적, 커스텀 커리큘럼) 동시 검증
- **고려 중:** 프리미엄 XR 콘텐츠 또는 고급 트랙의 개인 구독 모델

## 6. Scope & Timeline

### What needs to be true to test our hypothesis?

고전물리 트랙의 핵심 3개 모듈(운동과 힘, 에너지와 일, 파동)이 브라우저에서 60fps로 동작하고, 학생이 물체를 던지거나 힘의 크기·방향을 조절하며 물리 법칙을 즉시 체감할 수 있어야 한다. 롤러코스터를 직접 설계하고 에너지 변환을 관찰하는 수준의 인터랙션 품질이 필요하다.

### Rough timeline

| Phase | Duration | Key Activities |
|-------|----------|---------------|
| Discovery | 2-3주 | 기술 스파이크 (3D 렌더링 + 물리엔진), 교사 인터뷰 5명, 물체 던지기 프로토타입 |
| Phase 1: MVP | 2-3개월 | 고전물리 핵심 모듈 3개 제작 + B2B 파일럿 + 출시 |
| Phase 2: 화학 | 3-4개월 | 화학 트랙 (PubChem 연동, 분자 조작) |
| Phase 3: 우주과학 | 3-4개월 | 우주과학 트랙, 대중 시장 확장 |
| Phase 4: 양자역학 | 3-4개월 | 양자역학 트랙 (이중슬릿, 블로흐 구), 바이럴 마케팅 |

## 8. Risks & Dependencies

- **Technical:** 브라우저 XR 지원 수준이 브라우저별로 다름. Safari 제한적
- **Technical:** 복잡한 물리 시뮬레이션(N-body, 파동함수)의 브라우저 성능 한계
- **Business:** PhET 등 무료 대안이 존재하므로 차별화된 "와우" 없이는 전환 어려움
- **Market:** B2B 교육 시장 진입에는 교육부/학교 의사결정 사이클이 길 수 있음
- **Dependencies:** PDB, NASA API 등 외부 데이터 소스의 안정성과 라이선스

## 1. Problem

### What problem are we solving?

과학 교육에는 세 겹의 문제가 겹쳐 있다.

**(1) 차원의 문제 — 3D 개념을 2D 매체로 전달하는 한계**
분자 구조, 전자 오비탈, 파동 간섭, 궤도 역학 등 본질적으로 3차원인 개념이 2D 교과서와 정적 다이어그램으로 전달된다. 학생은 머릿속에서 3D를 재구성해야 하며, 이 인지 부담이 이해를 가로막는다.

**(2) 순서의 문제 — 수식이 직관보다 먼저 온다**
수식이 직관보다 먼저 제시되는 역순 학습 구조에서, 수학에 약한 학생은 과학 자체를 포기한다. "F=ma를 외웠지만 힘과 가속도의 관계가 어떤 느낌인지 모른다"는 것이 전형적인 증상이다.

**(3) 수동성의 문제 — "만질 수 있다"와 "배운다"는 다르다**
PhET처럼 인터랙티브 시뮬레이션이 있어도, 슬라이더를 만지작거리다 나가는 것이 현실이다. 능동적 사고를 유도하는 구조가 없으면 "만지는 교육"도 "보는 교육"과 다르지 않다. 학습 과학 연구에 따르면, 진짜 개념 변화(conceptual change)는 **"내 예측이 틀렸다"는 인지적 갈등(cognitive conflict)** 순간에 발생한다. 현재 어떤 도구도 이 순간을 체계적으로 설계하지 않는다.

**문제의 크기:**
- OECD PISA 2022 기준, 한국 학생의 과학 흥미도는 OECD 평균 이하. 한국 중고등학생 약 270만 명이 과학 교과를 수강하지만, 과학 진로 선택 비율은 지속 하락 추세
- 양자컴퓨팅, 생체분자, 우주산업 등 과학 리터러시 수요가 급증하지만, 기초 개념에 대한 직관적 이해 도구가 부재
- 기존 도구는 2D 위주(PhET), 고가 설치형($50K+/학교, Labster), 수동적 시청(YouTube), 문제풀이 중심(Brilliant, Khan Academy)
- 3D 체험 + 능동적 사고 유도 + 학습 동기 엔진을 통합한 도구가 존재하지 않음

현재 어떤 도구도 **"예측하게 만들고 → 3D로 실험하게 하고 → 틀린 지점에서 개념을 발견하게 하는"** 구조를 제공하지 않는다.

### Why now?

- **브라우저 3D 기술 성숙:** WebGPU 표준화, Three.js/R3F 생태계 성숙으로 실시간 3D + 물리 시뮬레이션이 실용적 수준에 도달
- **WebXR 표준화:** 별도 앱 없이 브라우저에서 XR 체험 가능. 동일 코드베이스로 3D → XR 점진적 전환
- **XR 기기 보급 확대:** Meta Quest 3, Apple Vision Pro 등 소비자 기기 확산
- **AI 적응형 학습:** LLM/ML 발전으로 사용자 수준에 맞는 적응형 난이도 조절이 기술적으로 가능
- **게이미피케이션 교육의 검증:** Duolingo가 언어 교육 시장을 재정의한 것처럼, 과학 교육에도 같은 기회가 존재하지만 아직 3D 체험과 결합한 사례가 없음
- **오픈 과학 데이터:** PDB, PubChem, NASA Open API 등 풍부한 공개 데이터

## 2. Proposed Direction

### Core insight

기존 과학 교육 도구의 핵심 문제는 "3D가 아니라서"도 아니고 "만질 수 없어서"도 아니다. **"생각하지 않고 만지기 때문"**이다.

PhysPlay의 첫 번째 베팅: **"예측 → 3D 실험 → 발견"이라는 루프를 게임의 코어 메카닉으로 만들면, 학습 동기와 학습 효과를 동시에 해결할 수 있다.**

두 번째 베팅: **통합 커리큘럼**. "달에서 공을 던졌더니 예상보다 멀리 갔네?" → "목성에서는?" → "중력이 빛도 휘게 한다면?" → 고전물리에서 양자역학까지 호기심의 사슬이 자연스럽게 연결되는 경험.

### Core Loop: 예측 → 실험 → 발견

```
┌─────────────────────────────────────────────┐
│                                             │
│   🎯 PREDICT                                │
│   "이 공을 달에서 던지면 어디에 떨어질까?"      │
│   → 사용자가 궤적을 그리거나 선택지를 고른다    │
│                                             │
│           ↓                                 │
│                                             │
│   🧪 PLAY                                   │
│   내 앞의 실험 대상을 직접 손으로 조작한다      │
│   웹: 마우스/터치 | XR: 핸드 트래킹            │
│                                             │
│           ↓                                 │
│                                             │
│   💡 DISCOVER                                │
│   예측 vs 결과 비교                           │
│   ✗ 틀렸다면 → "왜?" → 개념 설명 + 변수 조절  │
│   ✓ 맞았다면 → 변수 추가 → 더 어려운 예측     │
│                                             │
│           ↓                                 │
│                                             │
│   🔄 NEXT CHALLENGE                          │
│   AI가 수준에 맞는 다음 챌린지를 제시           │
│   호기심이 다음 단계의 동기가 된다              │
│                                             │
└─────────────────────────────────────────────┘
```

이 루프가 동기를 만드는 원리: (1) 예측은 참여를 강제한다 — "내 예측"이 걸려 있으면 결과에 관심을 갖는다. (2) 틀림은 호기심을 만든다 — "왜 내 예상과 다르지?" (3) 맞음은 자신감을 만든다 — 더 어려운 도전 동기. (4) 연결은 탐험을 만든다 — "달에서 맞았으니, 블랙홀 근처에서는?"

### 핵심 설계 결정

**God Hand 인터랙션 — "나는 실험자이고, 실험 대상은 내 앞에 있다"**
캐릭터/아바타 없이 1인칭 테이블탑 조작. 태양계를 축소해 놓고 돌리거나, 분자를 확대해 손으로 조립한다. 이 프레임이면 원자부터 은하까지 스케일이 자유롭고, 웹(마우스/터치)에서 XR(핸드 트래킹)까지 멘탈 모델이 동일하다. 캐릭터 개발 복잡도도 제거된다.

**엔진 기반 콘텐츠 — "24개 모듈 수동 제작"의 대안**
"4트랙 × 6모듈 = 24개 고정 콘텐츠"의 교과서식 구조 대신, 시뮬레이션 엔진(Layer 1) × 변수 조합 챌린지(Layer 2) × 공간 테마(Layer 3)의 3레이어 구조를 채택한다. 투사체 엔진 하나로 "지구에서 45도" → "달에서" → "공기저항 ON" → "목성에서" 같은 무한 챌린지가 생성된다. 콘텐츠는 가로(새 공간 해금), 세로(기존 공간에 스테이션 추가), 깊이(기존 엔진에 챌린지 추가) 세 방향으로 확장된다.

**스크린 구성 — 3D + 2D 혼합**
모든 화면이 3D일 필요는 없다. Play 단계만 3D 뷰포트, Predict/Discover는 3D 위 2D 오버레이, 허브/설정은 순수 2D로 리소스 절약. 모바일에서 허브 화면에 3D를 돌리면 배터리/발열 문제가 생기기 때문.

**모바일 퍼스트**
Primary 타깃(한국 중고등학생)이 거의 100% 스마트폰으로 접속. 터치 제스처(스와이프 던지기, 손가락 드로잉, 핀치 줌)가 기본 인터랙션이며, 데스크톱/XR은 같은 엔진에서 입력만 매핑한다.

**첫 30초 — 온보딩이 곧 첫 번째 챌린지**
URL을 열면 회원가입/트랙 선택 없이 바로 "공 던지기" 챌린지. 30초 안에 예측→실험→발견 한 바퀴를 체험한 후, 연구소 허브가 열리며 전체 공간 맵이 등장한다.

**UGC 비전 — 유저가 만드는 과학 실험**
챌린지의 본질이 "엔진 + 변수 조합 + 예측 질문"이라는 데이터(JSON)이기 때문에, AI가 챌린지를 만드는 것과 유저가 만드는 것은 구조적으로 동일하다. Phase 1부터 데이터 구조를 설계하고, Phase 3부터 교사 챌린지 에디터, Phase 5+에서 커뮤니티 공간 제작으로 점진적 개방.

**AI — 적응형 경험 엔진 (디렉터 AI)**
풀 AI 챗봇이 아니라, 사용자의 예측 패턴과 정확도를 분석하여 변수 조합을 선택하고, 난이도와 설명 깊이를 조절하는 게임 디렉터. Phase 1은 규칙 기반, Phase 2+에서 ML 전환.

### Differentiation

| 차별점 | 설명 |
|--------|------|
| **예측-실험-발견 루프** | "만지는 교육"을 넘어 "생각하는 교육". 인지적 갈등을 핵심 메카닉으로 설계한 유일한 과학 교육 도구 |
| **God Hand 인터랙션** | 1인칭 실험자 + 테이블탑 조작. 웹에서 XR까지 동일 멘탈 모델 |
| **엔진 기반 무한 챌린지** | 시뮬레이션 엔진 × 변수 조합으로 콘텐츠가 자동 생성 |
| **확장하는 연구소** | 새 공간 해금 + 기존 공간 확장의 이중 성장. 소유감과 탐험 동기 |
| **적응형 AI 엔진** | 사용자 수준에 맞게 챌린지를 생성하고, 난이도와 설명 깊이를 자동 조절 |
| **3D-native + XR Enhanced** | 처음부터 3D로 설계하고, XR로 체감을 강화. 점진적 몰입 |
| **제로 장벽 접근** | URL 하나로 시작, 설치 없음, 기본 무료 |

**핵심 포지션:** PhET의 접근성 + Labster의 3D 몰입 + Brilliant의 인터랙션 + **Duolingo의 학습 동기 엔진**을 브라우저에서 무료로 통합한다.

### Non-Goals

- 기존 LMS 대체 — LMS와 병행 사용
- 공인 교육과정/학점 — 체험과 이해가 목적
- 고정밀 연구용 시뮬레이션 — 교육적 정확성과 직관이 우선
- 네이티브 앱 — 웹 기반, URL 하나로 접근
- AI 튜터/챗봇 — AI는 적응형 경험 엔진
- 포인트만 쌓는 얕은 게이미피케이션 — 핵심 동기는 인지적 갈등에서 오는 호기심
- 3인칭 RPG / 아바타 — 1인칭 실험자 + God Hand
- 24개 모듈 수동 제작 — 엔진 기반 무한 챌린지 생성

상세 설계는 [PRD](./prd.md) 참조.

## 3. Assumptions, Risks & Dependencies

| Assumption / Risk | Type | Impact | How We'll Address It |
|-------------------|------|--------|---------------------|
| "예측→실험→발견" 루프가 자유 탐색보다 학습 효과가 높다 | Value | High | A/B 테스트: 자유 탐색 vs 예측 루프 모드, 사전/사후 이해도 비교 (n=20) |
| 게이미피케이션 구조가 과학 교육의 진지함을 해치지 않는다 | Value | High | Discovery 중 교사 5명 인터뷰 + "게임 같은 과학 수업"에 대한 인식 확인 |
| 예측 참여가 학습 동기(다음 모듈 진행)를 실제로 높인다 | Value | High | Phase 1에서 모듈 완주율 측정 (목표: 40%) |
| 3D 인터랙션이 2D 다이어그램보다 개념 이해도를 높인다 | Value | High | 사용자 테스트에서 사전/사후 이해도 비교 |
| 엔진 기반 변수 조합 챌린지가 수동 제작 챌린지만큼 교육 효과가 있다 | Value | Medium | Phase 1에서 수동 설계 챌린지 vs 변수 조합 챌린지 비교 |
| 적응형 난이도 AI가 1인 개발로 구현 가능한 수준이다 | Feasibility | Medium | Phase 1은 규칙 기반 분기로 시작, AI 모델은 Phase 2+에서 도입 |
| 브라우저에서 물리 시뮬레이션을 60fps로 구동할 수 있다 | Feasibility | High | 물체 던지기 + 롤러코스터 프로토타입 eng spike (2주) |
| God Hand 테이블탑 모델이 웹↔XR 전환에서 일관된 경험을 제공한다 | Feasibility | Medium | Discovery에서 물체 던지기 프로토타입으로 웹/XR 양쪽 테스트 |
| 3D/XR 동일 코드베이스로 Progressive XR 구조를 유지할 수 있다 | Feasibility | Medium | XR 모드 전환 프로토타입 검증 (Discovery) |
| 무료 웹 접근만으로 유의미한 트래픽을 확보할 수 있다 | Viability | High | 출시 후 1개월 내 10K UV 달성 여부로 판단 |
| B2B 교육 시장의 의사결정 사이클이 짧지 않을 수 있다 | Viability | Medium | 무료 B2C + 교사 개인 단위 접근 병행 |
| XR 브라우저 지원 수준이 브라우저별로 다름 | Dependency | Medium | 3D First — XR은 보너스, 3D가 기본 |
| PDB, NASA API 등 외부 데이터 소스의 안정성과 라이선스 | Dependency | Low | Phase 2+ 진입 전 라이선스 검토, 캐싱/폴백 구현 |

**Counter-metrics:**
- 물리 시뮬레이션 성능 — mid-range 디바이스에서 60fps 유지
- 예측 스킵율 — 예측 단계를 건너뛰는 비율이 30% 미만이어야 함 (높으면 예측 UX가 번거롭다는 신호)
- 챌린지 반복감 — 변수 조합 챌린지가 "같은 걸 또 하는 느낌"을 주지 않아야 함 (정성 피드백으로 확인)

## 5. Market & Pricing

**핵심 경쟁 포지션:** PhET의 접근성 + Labster의 3D 몰입 + Brilliant의 인터랙션 + Duolingo의 학습 동기 엔진

**Pricing Signal:**
- **B2C:** 기본 무료 (모든 공간). 무료로 "와우" 경험을 만들어 바이럴을 유도
- **B2B:** 교사/학교 대상 프리미엄 — 교사 대시보드(학생별 예측 패턴/이해도 분석), 커스텀 챌린지 구성, 학급 관리
- **검증 필요:** B2B 라이선스 가격대, 교사 개인 vs 기관 단위 구매 의향
- **장기 고려:** 프리미엄 XR 콘텐츠, 고급 적응형 AI 기능의 개인 구독 모델

## 6. Scope & Timeline

### Product scope

**시뮬레이션 엔진 기반의 확장 가능한 과학 게이미피케이션 플랫폼.** 4개 공간(역학 실험실, 분자 실험실, 우주 관측소, 양자 연구소)이 점진적으로 해금되며, 각 공간 안에서 스테이션이 추가되고, 각 스테이션에서 챌린지가 변수 조합으로 생성된다. 인터랙션은 God Hand 모델로 통일, 브라우저에서 XR까지 일관된 경험을 제공한다. 장기적으로 유저가 직접 챌린지와 공간을 만드는 UGC 생태계로 진화한다.

### Rough timeline

| Phase | Duration | Description |
|-------|----------|-------------|
| Discovery | 2-3주 | 투사체 엔진 프로토타입 + God Hand + 코어 루프 통합 검증, 교사 인터뷰 |
| Phase 1 (역학 실험실) | 2-3개월 | 엔진 3개 + 첫 공간 출시 |
| Phase 2 (역학 확장 + AI) | 1-2개월 | 역학 실험실 세로 확장 + 적응형 AI v1 |
| Phase 3 (분자 실험실) | 3-4개월 | 가로 확장(새 공간) + 계정/B2B 파일럿 |
| Phase 4 (우주 관측소) | 3-4개월 | 가로 확장 + NASA API 연동 |
| Phase 5 (양자 연구소) | 3-4개월 | 가로 확장 + 바이럴 정점 |
| Phase 6+ | Ongoing | 세로/깊이 확장 + 신규 공간 + UGC |

상세 Phase 계획은 [PRD](./prd.md) 참조.

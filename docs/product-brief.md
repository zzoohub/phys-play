# PhysPlay — Product Brief

**Author:** — | **Date:** 2026-03-02 | **Status:** Discovery
**Tagline:** 예측하고, 실험하고, 발견하라 — 3D 과학 게이미피케이션 플랫폼

---

## 1. Problem

과학 교육에는 세 겹의 문제가 겹쳐 있다.

**(1) 차원의 문제 — 3D 개념을 2D 매체로 전달하는 한계**
분자 구조, 전자 오비탈, 파동 간섭, 궤도 역학 등 본질적으로 3차원인 개념이 2D 교과서와 정적 다이어그램으로 전달된다. 학생은 머릿속에서 3D를 재구성해야 하며, 이 인지 부담이 이해를 가로막는다.

**(2) 순서의 문제 — 수식이 직관보다 먼저 온다**
수식이 직관보다 먼저 제시되는 역순 학습 구조에서, 수학에 약한 학생은 과학 자체를 포기한다. "F=ma를 외웠지만 힘과 가속도의 관계가 어떤 느낌인지 모른다"는 것이 전형적인 증상이다.

**(3) 수동성의 문제 — "만질 수 있다"와 "배운다"는 다르다**
PhET처럼 인터랙티브 시뮬레이션이 있어도, 슬라이더를 만지작거리다 나가는 것이 현실이다. 능동적 사고를 유도하는 구조가 없으면 "만지는 교육"도 "보는 교육"과 다르지 않다. 학습 과학 연구에 따르면, 진짜 개념 변화(conceptual change)는 **"내 예측이 틀렸다"는 인지적 갈등(cognitive conflict)** 순간에 발생한다. 현재 어떤 도구도 이 순간을 체계적으로 설계하지 않는다.

**문제의 크기:**
- OECD PISA 2022 기준, 한국 학생의 과학 흥미도는 OECD 평균 이하. 과학 진로 선택 비율은 지속 하락 추세
- 기존 도구는 2D 위주(PhET), 고가 설치형($50K+/학교, Labster), 수동적 시청(YouTube), 문제풀이 중심(Brilliant, Khan Academy)
- 3D 체험 + 능동적 사고 유도 + 학습 동기 엔진을 통합한 도구가 존재하지 않음

**Who has this problem?**

- **"과학이 지루한 서연" (10세, 초4)** — "내가 직접 만져보고, 맞추고, 틀리면 왜 틀렸는지 바로 볼 수 있으면 과학 시간이 게임처럼 재밌을 것 같다."
- **"수학이 싫은 민준" (16세, 고1)** — "내가 예측하고, 실험하고, 내 생각이 맞는지 틀렸는지 바로 확인할 수 있으면 — 다음 단계가 궁금해서 계속하게 될 것 같다."
- **"커리어 전환 중인 지영" (29세, 개발자)** — "틀려도 괜찮은 환경에서 직접 해보면서 양자역학 직관을 잡고 싶다."

**How do they solve it today?**

| 현재 도구 | 강점 | 한계 | 빠진 것 |
|-----------|------|------|---------|
| 교과서 | 체계적 커리큘럼 | 2D 정적, 수식 우선 | 체험, 동기 |
| YouTube | 뛰어난 시각화, 무료 | 수동적 시청 | 능동적 사고, 피드백 |
| PhET Simulations | 인터랙티브, 무료 | 대부분 2D, 목표 없는 탐색 | 학습 동기, 구조화된 경험 |
| Brilliant.org | 인터랙티브 문제풀이 | 3D 제한적, 문제풀이 중심 | 3D 체험, 실험적 발견 |
| Labster | VR 실험실 | 고가($50K+/학교), 설치형 | 접근성, 적응형 경험 |

지금이 가능한 이유: WebGPU 표준화와 Three.js/R3F 생태계 성숙으로 브라우저에서 실시간 3D 물리 시뮬레이션이 실용적 수준에 도달했고, WebXR로 동일 코드베이스에서 XR 전환이 가능하다. Duolingo가 언어 교육을 재정의한 것처럼 3D 체험과 결합한 과학 교육 게이미피케이션은 아직 없다.

## 2. Proposed Direction

### Core insight

기존 과학 교육 도구의 핵심 문제는 "3D가 아니라서"도 아니고 "만질 수 없어서"도 아니다. **"생각하지 않고 만지기 때문"**이다.

### Core Loop: 예측 → 실험 → 발견

```
PREDICT  →  PLAY  →  DISCOVER  →  NEXT CHALLENGE
 예측을       3D로      예측 vs 결과     AI가 수준에 맞는
 먼저 한다    실험한다   비교 → 개념 발견  다음 챌린지 제시
```

이 방향이 성립하려면: (1) 예측-실험-발견 루프가 자유 탐색보다 학습 효과와 동기를 높여야 한다 — 예측 참여율 70%, 완주율 40% 달성으로 검증. (2) 초등학생부터 성인까지 동일 코어 루프로 커버할 수 있어야 한다 — 적응형 난이도/설명 깊이 조절로 검증. (3) 브라우저에서 물리 시뮬레이션을 60fps로 구동할 수 있어야 한다 — 투사체 프로토타입 eng spike로 검증.

### 핵심 설계 결정

- **God Hand 인터랙션** — 캐릭터/아바타 없이 1인칭 테이블탑 조작. 원자부터 은하까지 스케일이 자유롭고, 웹에서 XR까지 멘탈 모델이 동일
- **엔진 기반 콘텐츠** — 시뮬레이션 엔진 × 변수 조합 × 공간 테마의 3레이어 구조. 콘텐츠는 가로(새 공간), 세로(스테이션 추가), 깊이(챌린지 추가) 세 방향으로 확장
- **풀스크린 3D + HUD** — 실험실은 전체 3D 뷰포트, UI는 HUD 오버레이. 허브/설정 등 바깥 화면은 2D
- **첫 30초 = 첫 챌린지** — URL 열면 회원가입 없이 바로 "공 던지기". 30초 안에 코어 루프 한 바퀴 체험
- **PC 퍼스트, 모바일 서포트** — 넓은 화면에서 최적 경험, 터치 제스처로 모바일 지원, XR은 입력만 매핑

### Differentiation

**핵심 포지션:** PhET의 접근성 + Labster의 3D 몰입 + Brilliant의 인터랙션 + **Duolingo의 학습 동기 엔진**을 브라우저에서 무료로 통합.

| 차별점 | 설명 |
|--------|------|
| 예측-실험-발견 루프 | 인지적 갈등을 핵심 메카닉으로 설계한 유일한 과학 교육 도구 |
| God Hand + Must 3D | 2D로는 불가능한 체험이 기본. 웹에서 XR까지 동일 멘탈 모델 |
| 엔진 기반 무한 챌린지 | 시뮬레이션 엔진 × 변수 조합으로 콘텐츠 자동 생성 |
| 제로 장벽 접근 | URL 하나로 시작, 설치 없음, 기본 무료 |

### Non-Goals

- B2B / 학교·기관 판매 — B2C 개인 사용자 경험에 집중
- 공인 교육과정/학점 — 체험과 이해가 목적
- 네이티브 앱 — 웹 기반, URL 하나로 접근
- AI 튜터/챗봇 — AI는 적응형 경험 엔진 (디렉터 AI)
- 3인칭 RPG / 아바타 — 1인칭 실험자 + God Hand
- 24개 모듈 수동 제작 — 엔진 기반 무한 챌린지 생성

## 3. Success Criteria

| Metric | Target | How Measured |
|--------|--------|-------------|
| **예측 참여율** | 스테이션 진입자의 70%가 1회 이상 예측 참여 | Event tracking |
| **스테이션 완주율** | 진입자의 40%가 챌린지 시퀀스 완료 | Event tracking |
| **월간 방문자 (UV)** | 10,000 (출시 첫 달) | Web analytics |

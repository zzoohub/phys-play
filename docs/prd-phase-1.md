## 1. Phase Goal

**가설:** "수학에 약한 중고등학생이 브라우저에서 3D 시뮬레이션의 변수를 직접 조작하면, 수식 없이도 고전물리 핵심 법칙(뉴턴 역학, 에너지 보존, 파동 중첩)을 직관적으로 이해할 수 있다."

고전물리 트랙의 핵심 3개 모듈(운동과 힘, 에너지와 일, 파동)을 브라우저에서 60fps로 구동하고, 이 가설을 사용자 테스트(n=20, 사전/사후 이해도 20%+ 향상)와 행동 데이터(세션 체류 3분+, 인터랙션율 60%+)로 검증한다.

---

## 2. Scoping Rationale

**왜 고전물리 3개 모듈인가:**

1. **사용자 풀 최대화:** 모든 중고등 교육과정에 포함되는 범용 주제
2. **인터랙션 품질 검증:** 물체 던지기(1-1), 롤러코스터 설계(1-2), 파동 합성(1-3)은 직관적이고 재미있는 인터랙션으로, "만지는 교육"의 핵심 가치를 가장 잘 보여줌
3. **외부 의존 없음:** PDB, NASA API 등 외부 데이터 의존 없이 순수 물리 시뮬레이션만으로 구현 가능 → 빠른 출시
4. **기술 기반 검증:** 3D 렌더링, 물리 엔진, 인터랙션 UI, 반응형 설계 등 전체 트랙에 공통으로 필요한 기술 스택을 한 번에 검증
5. **후속 Phase 활용:** Phase 1에서 확보한 사용자 행동 데이터(체류 시간, 인터랙션 패턴, 디바이스별 성능)가 화학/우주/양자 트랙 설계에 직접 활용됨

**왜 3개인가 (6개가 아닌):**

1-4(소리와 빛), 1-5(전기와 자기), 1-6(전자기파)는 인터랙션 복잡도가 높고(회로 조립, 렌즈 광선추적 등) 개발 기간이 길다. 핵심 3개 모듈로 가치 가설을 빠르게 검증한 뒤, Phase 2에서 나머지를 추가한다.

## 5. Phase 1 User Journeys

### Journey A: 학생의 첫 방문

1. 학생이 URL(또는 SNS/교사 공유 링크)로 접근한다
2. 랜딩 페이지에서 PhysPlay의 가치와 고전물리 3개 모듈을 소개한다
   - 전체 비전(4개 트랙)도 "Coming Soon"으로 조망 가능 (P1)
   - 재방문 시: IndexedDB 기반으로 이전 체험 모듈과 관련된 추천 표시 (예: "운동과 힘을 체험했습니다. 에너지와 일도 해보세요")
3. 학생이 모듈을 선택한다 (예: "운동과 힘")
4. 모듈 진입 시 학습 목표가 간략히 표시된다 (P1)
5. 3D 시뮬레이션이 로드된다
   - 로딩 중 프로그레스 표시
   - If 로드 실패 → 에러 메시지 + 재시도 옵션 + 호환 모드 자동 전환
6. 첫 인터랙션 유도가 표시된다 ("슬라이더를 움직여 힘의 크기를 바꿔보세요")
7. 학생이 변수를 조작하며 시뮬레이션을 체험한다
8. (P1) 체험 후 "이 현상의 물리 법칙 보기" 옵션이 표시된다. 탭하면 관련 수식과 법칙 설명이 나타난다
9. 모듈 하단에 다른 모듈 추천이 표시된다
10. 학생이 다른 모듈로 이동하거나 세션을 종료한다

**Phase 1 vs Full 차이:** 계정 없이 익명 접근. 학습 진도 로컬 저장 (IndexedDB, 브라우저 재시작 후에도 유지). 3개 모듈만 체험 가능.

### Journey B: 교사의 수업 활용

1. 교사가 PhysPlay URL에 접근한다
2. "운동과 힘" 모듈을 선택한다
3. 프로젝터에 시뮬레이션을 띄우고 실시간으로 변수를 조작하며 설명한다
4. 학생들에게 URL을 공유하여 각자 디바이스에서 직접 체험하게 한다

**에러 상태:** 학생 디바이스에서 시뮬레이션 로드 실패 → 브라우저 호환성 안내 + 품질 자동 조정 시도.

**Phase 1 vs Full 차이:** 교사 대시보드 없음. 학급 관리 없음. URL 공유만 가능.

### Journey C: 롤러코스터 설계 체험 (킬러 콘텐츠)

1. 학생이 "에너지와 일" 모듈에 진입한다
2. 빈 캔버스에서 롤러코스터 트랙을 마우스/터치로 그린다
3. "시작" 버튼을 누르면 물체가 트랙을 따라 이동한다
4. 운동에너지/위치에너지 바가 실시간으로 변한다
5. 학생이 마찰을 ON하면 에너지 손실이 발생하고, 물체가 끝까지 도달하지 못한다
6. 학생이 트랙을 수정하여 다시 시도한다 ("마찰이 있어도 끝까지 가려면?")
7. 반복 실험을 통해 에너지 보존 법칙을 직관적으로 체득한다

**드롭오프 위험:** Step 2 (트랙 그리기). 그리기 인터페이스가 직관적이지 않으면 좌절한다. 기본 트랙 프리셋 제공 + 수정 옵션이 필요할 수 있다.

## 6. Phase 1 Success Criteria

| Goal | Metric | Counter-metric | Phase 1 Target | Full Product Target |
|------|--------|----------------|------------|---------------------|
| 사용자 확보 | 월간 방문자 (UV) | CAC (유입 채널별 비용) | 10,000 | 50,000+ |
| 체험 깊이 | 평균 세션 체류 시간 | 이탈률 (bounce rate) | 3분 | 5분+ |
| 인터랙션 참여 | 시뮬레이션 인터랙션율 | 시뮬레이션 로드 실패율 | 60% | 75% |
| 콘텐츠 탐색 | 2개 이상 모듈 방문 비율 | 세션당 페이지 에러율 | 30% | 50% |
| B2B 초기 접근 | 교사 관심 표명 (이메일 수집) | 이메일 수집 후 이탈률 | 50명 | 파일럿 MOU 10곳 (Phase 3+) |
| 성능 안정성 | mid-range 60fps 유지율 | 디바이스별 크래시율 | 95% | 95% |
| 교육적 가치 | 사용자 테스트 이해도 향상 | 테스트 참여 완료율 | 사전 대비 20%+ 향상 (n=20) | 지속적 개선 |
| 교육적 검증 | 교사/교육자 피드백 인터뷰 | — | 5명 이상 인터뷰 완료 | 10명 이상 (출시 후 3개월) |

## 1. Problem / Opportunity

### The Problem

과학의 핵심 개념 — 분자의 3차원 구조, 파동의 간섭 패턴, 궤도 역학, 양자 확률 — 은 본질적으로 3차원이다. 그러나 학생들은 2D 교과서, 정적 다이어그램, 수동적 영상으로 이를 학습한다. 수식이 직관보다 먼저 제시되는 역순 학습 구조에서, 수학에 약한 학생은 과학 자체를 포기한다.

**문제의 규모:**
- OECD PISA 2022 기준, 한국 학생의 과학 흥미도는 OECD 평균 이하. 한국 중고등학생 약 270만 명이 과학 교과를 수강하지만, 과학 진로 선택 비율은 지속 하락 추세
- 양자컴퓨팅, 생체분자, 우주산업 등 과학 리터러시 수요가 급증하지만, 기초 개념에 대한 직관적 이해 도구가 부재
- 기존 3D 교육 도구는 대부분 2D 위주(PhET), 고가 설치형($50K+/학교, Labster), 또는 단일 주제 한정
- 물리 → 화학 → 우주 → 양자를 관통하는 통합 체험형 학습 경험이 존재하지 않음

**현재 해결 방식과 한계:**
- **교과서** — 3D 개념을 2D로 이해해야 하는 근본적 한계
- **YouTube (3Blue1Brown 등)** — 뛰어난 시각화이나 조작 불가, 수동적 시청
- **PhET Simulations** — 방대하지만 대부분 2D, 콘텐츠 분산, 통합 학습 경로 없음
- **Khan Academy** — 영상+문제풀이 중심, 체험형이 아님
- **Labster** — VR 실험실이나 고가($50K+/학교), 설치형, 접근성 제한

### Why Now

- **브라우저 3D 기술 성숙:** 브라우저에서 고품질 3D 렌더링과 실시간 물리 시뮬레이션이 실용적 수준에 도달. 별도 앱 설치 없이 URL 하나로 접근 가능
- **XR 기기 보급:** Meta Quest 3, Apple Vision Pro 등 소비자 기기 확산. 동일 콘텐츠를 3D → XR로 점진적 전환 가능
- **테크 관심층 급증:** 양자컴퓨팅, AI, 우주산업 관심 폭증으로 과학 기초 리터러시 수요 증가
- **오픈 과학 데이터:** PDB, PubChem, NASA Open API 등 콘텐츠 기반이 되는 공개 데이터가 풍부

| Labster | VR 랩, 강력한 B2B | 고가($50K+/학교), 설치형 | 무료 웹 접근, URL 하나로 시작 |

**핵심 포지션:** PhET의 접근성 + Labster의 3D 몰입 + Brilliant의 인터랙션을 브라우저에서 무료로 통합한다.

## 2. Target Users & Use Cases

### Personas

**Primary — "수학이 싫은 민준" (16세, 고1)**

과학은 재밌을 것 같지만 수식이 나오면 흥미를 잃는다. F=ma를 외웠지만 힘과 가속도의 관계가 어떤 "느낌"인지 모른다. YouTube에서 양자역학 영상을 봤지만 "결국 수식 없이는 이해 못 한다"는 말에 좌절했다.

> JTBD: "과학 시간에 배운 개념이 뭘 의미하는지, 직접 만져보면서 내 몸으로 이해하고 싶다. 수식 대신 체험으로 시작해서 나중에 수식이 왜 그런 모양인지 이해하고 싶다."

**Secondary — "커리어 전환 중인 지영" (29세, 개발자)**

양자컴퓨팅에 관심이 생겼지만 물리학 배경이 없다. 블로흐 구를 직접 회전시키며 큐빗 상태를 조작하고 싶다. 분야별로 도구를 찾아 헤매지 않고 하나의 플랫폼에서 탐색하고 싶다.

> JTBD: "양자컴퓨팅 논문을 읽으려면 양자역학 기초가 필요한데, 교과서로 독학하기엔 벽이 너무 높다. 3D로 조작하면서 직관을 잡을 수 있는 도구가 필요하다."

**Secondary — "시각적 교보재가 필요한 박 선생님" (35세, 물리 교사)**

수업에서 힘의 벡터, 에너지 변환, 분자 구조를 설명할 때 2D 자료의 한계를 느낀다. 학생들이 직접 변수를 조작하며 과학 법칙을 체험할 수 있는 도구가 필요하다.

> JTBD: "수업 중에 학생들이 직접 변수를 바꿔가며 물리 법칙을 관찰할 수 있는 교보재가 필요하다. 설치나 비용 없이 URL만 공유하면 바로 사용할 수 있어야 한다."

### Core Use Cases

**UC-1. 자기 주도 탐험 (Primary)**
학생이 흥미 있는 주제를 선택하고, 3D 시뮬레이션을 조작하며 과학 개념을 체험한다. 수식 없이 슬라이더, 드래그, 토글로 변수를 조절하면 결과가 즉시 변한다.

**UC-2. 추천 학습 경로 따라가기**
진입 동기에 맞는 경로를 선택하여 안내를 받는다. "흥미 우선"(우주 → 양자 → 화학 → 고전), "기초 탄탄"(고전 → 화학 → 우주 → 양자), "하이라이트"(킬러 모듈 맛보기 → 관심 트랙 탐색) 중 선택한다.

**UC-3. 수업 보조 도구로 활용 (교사)**
교사가 수업 중 특정 모듈을 열어 실시간으로 시연하거나, 학생들에게 URL을 공유해 직접 체험하게 한다.

**UC-4. XR 몰입 체험 (보너스)**
XR 기기가 있는 사용자가 동일 콘텐츠를 공간 몰입 모드로 전환하여, 스케일 체감과 공간 인터랙션을 경험한다.

## 3. Proposed Solution

### Elevator Pitch

브라우저만 열면 바로 접근할 수 있는 3D 인터랙티브 과학 체험 플랫폼. 4개 과학 분야(고전물리, 화학, 우주과학, 양자역학) × 6개 모듈 = 24개 모듈을 관통하는 체계적 학습 경로를 제공한다. 수식 대신 인터랙션으로 직관을 먼저 형성하고, XR 기기가 있으면 몰입이 자동으로 강화된다.

### Core Value Propositions

1. **만지는 교육** — 보는 교육이 아니라 조작하는 교육. 슬라이더, 드래그, 토글로 변수를 직접 조절하면 결과가 즉시 변한다 → "수식 없이 와닿지 않는" 문제 해결
2. **통합 커리큘럼** — 고전물리의 파동 → 양자역학의 이중성, 화학의 분자 → 생체분자 등 분야를 관통하는 학습 경로 → "콘텐츠가 분산되어 있다"는 문제 해결
3. **제로 장벽 접근** — URL 하나로 시작, 설치 없음, 기본 무료 → "고가/설치형"이라는 접근성 문제 해결

### Mental Model

"과학 실험실을 브라우저 안에 넣었다." 물리적 실험실에서 실험 장치의 변수(힘의 크기, 온도, 각도)를 직접 조절하듯, 브라우저 안에서 3D 시뮬레이션의 파라미터를 실시간 조작한다. 하나의 실험실에 물리, 화학, 우주, 양자 섹션이 모두 있고, 자유롭게 이동할 수 있다.

3. 흥미 있는 트랙 또는 모듈을 선택한다
4. 모듈 진입 시 학습 목표와 핵심 개념 요약이 표시된다
5. 3D 시뮬레이션이 로드되고, 기본 인터랙션 안내(첫 방문 시 가이드)가 표시된다
6. 사용자가 슬라이더/토글/드래그로 변수를 조작하며 시뮬레이션을 체험한다
7. 모듈 하단에 "다음 모듈" 또는 "관련 모듈" 추천이 표시된다
8. 사용자가 다른 모듈로 이동하거나 세션을 종료한다

**드롭오프 위험:** Step 5 (시뮬레이션 로딩). 3D 콘텐츠 로딩이 느리면 이탈한다. 로딩 중에도 시각적 피드백(프로그레스, 스켈레톤)이 필요하다.

**드롭오프 위험:** Step 6 (첫 인터랙션). 무엇을 해야 할지 모르면 이탈한다. "이 슬라이더를 움직여보세요" 같은 명시적 첫 액션 유도가 필요하다.

### Journey 2: 추천 학습 경로

1. 사용자가 "추천 경로로 시작" 옵션을 선택한다
2. 3가지 경로(흥미 우선 / 기초 탄탄 / 하이라이트)와 각각의 설명이 표시된다
3. 사용자가 경로를 선택한다
   - 흥미 우선: 우주과학 트랙부터 안내
   - 기초 탄탄: 고전물리 트랙부터 안내
   - 하이라이트: 각 트랙의 킬러 모듈(1-2, 2-1, 3-2, 4-2) 순서로 안내
4. 각 모듈 완료 후 경로상 다음 모듈로 자동 안내된다
5. 진행 중 언제든 자유 탐험 모드로 전환하거나 경로를 변경할 수 있다

### Journey 3: 교사의 수업 활용

1. 교사가 수업 전에 특정 모듈의 URL을 확인한다
2. 수업 중 프로젝터/스크린에 해당 모듈을 띄운다
3. 교사가 실시간으로 변수를 조작하며 설명한다
4. 학생들에게 URL을 공유하여 각자 디바이스에서 직접 체험하게 한다
5. (Phase 3+) 교사 대시보드에서 학생들의 인터랙션 현황을 확인한다

### Journey 4: XR 전환 체험

1. XR 기기를 착용한 상태에서 플랫폼에 접근한다
2. 시스템이 XR 기기를 감지하고 "몰입 모드 전환" 옵션을 표시한다
3. 사용자가 전환하면 동일 시뮬레이션이 공간 몰입 모드로 렌더링된다
4. 핸드 트래킹으로 3D 객체를 직접 조작한다
5. 언제든 3D 모드로 복귀할 수 있다

## 7. Scope & Non-Goals

### In Scope

- 4개 과학 트랙(고전물리, 화학, 우주과학, 양자역학) × 6개 모듈 = 24개 모듈
- 브라우저 기반 3D 인터랙티브 시뮬레이션
- XR 전환 지원
- 다국어 지원 (en, es, pt-BR, id, ja, ko)
- 반응형 디자인 (데스크톱, 태블릿, 모바일)
- 라이트/다크 모드
- 랜딩 페이지
- B2B 교사 기능 (Phase 3+)
- 사용자 계정 및 학습 진도 저장 (Phase 3+)

### Out of Scope

| 항목 | 제외 사유 |
|------|-----------|
| LMS(학습관리시스템) 대체 | 과제 제출, 성적 관리 등은 기존 LMS와 병행. PhysPlay는 체험 도구 |
| 학점/자격증 부여 | 공인 교육과정이 아닌 체험과 이해가 목적 |
| 고정밀 연구용 시뮬레이션 | 교육적 정확성과 직관이 우선. 전문 연구자 대상이 아님 |
| 네이티브 앱 (iOS/Android) | 웹 기반으로 설치 없이 URL 접근. 네이티브 앱 필요성은 데이터로 판단 |
| 사용자 제작 콘텐츠(UGC) | 4개 트랙에 집중. 커리큘럼 외 자유 콘텐츠 업로드 미지원 |
| 실시간 멀티플레이어 | 네트워킹 복잡도 대비 교육적 가치 불확실. 추후 검토 |
| AI 튜터/챗봇 | 인터랙션 중심 제품 정체성에 집중. AI 보조는 별도 Discovery 필요 |

## 8. Assumptions, Constraints & Risks

### Assumptions

| Assumption | 검증 방법 |
|------------|-----------|
| 3D 인터랙션이 2D 다이어그램보다 개념 이해도를 높인다 | 사용자 테스트에서 사전/사후 이해도 비교 (n=20, 통제군 포함) |
| 수식 없는 접근이 교육 전문가에게도 받아들여진다 | Discovery 중 과학 교사 5명 인터뷰 + 피드백 수집 |
| 무료 웹 접근만으로 유의미한 트래픽을 확보할 수 있다 | 출시 후 1개월 내 10K UV 달성 여부로 판단 |
| Phase 1부터 B2B 파일럿으로 유료 모델을 검증할 수 있다 | 교사 파일럿 10곳 확보 후 전환율 측정 |

### Constraints

- 1인 개발 → 각 Phase별 작업량을 철저히 관리해야 함
- Phase 1 기간: 2~3개월 (Discovery 포함)
- 외부 데이터(PDB, NASA API) 라이선스를 Phase 3+ 진입 전 검토해야 함
- GDPR/PIPA 등 개인정보 규정은 계정 시스템 도입 시(Phase 3+) 준수해야 함

### Risks

| Risk | Severity | Likelihood | Mitigation |
|------|----------|------------|------------|
| 물리 시뮬레이션이 모바일 브라우저에서 60fps 미달 | High | Medium | 시뮬레이션 복잡도 제한 + 모바일용 경량 모드 자동 분기 |
| B2B 교육 시장 의사결정 사이클이 길어 매출 발생 지연 | High | Medium | 무료 B2C + 교사 개인 단위 접근 병행 |
| 첫 사용자 유입 채널이 불명확 | High | Medium | Product Hunt, 교사 커뮤니티, SNS, SEO 등 다채널 실험 |
| XR 브라우저 지원 수준이 브라우저별로 상이 | Medium | High | 3D First 철학 — XR은 보너스, 3D가 기본 |
| 교육적 정확성과 시뮬레이션 성능 간 트레이드오프 | Medium | Medium | 교육적 정확성 우선, 물리 정밀도는 교육 목적 수준으로 한정 |
| 외부 데이터 소스(PDB, NASA API) 불안정 또는 라이선스 변경 | Low | Low | 캐싱/폴백 구현, Phase 2+ 진입 전 라이선스 검토 |

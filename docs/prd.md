## 1. Problem / Opportunity

과학에서 가장 핵심적인 개념들 — 힘과 운동의 관계, 에너지 변환, 파동 현상 — 은 눈에 보이지 않는 물리량(벡터, 에너지, 진동)을 다룬다. 그러나 현재 교육은 2D 교과서, 정적 다이어그램, 수동적 영상에 의존한다.

**문제의 크기:**
- 한국 중고등학생 약 270만 명이 과학 교과를 수강하며, OECD PISA 2022 기준 한국 학생의 과학 흥미도는 OECD 평균 이하
- 기존 3D 교육 도구(PhET, Labster)는 2D 위주이거나 고가($50K+/학교) 설치형으로, 접근성과 인터랙션 모두 부족
- 학생들은 수식을 먼저 접하고 직관을 나중에 형성하는 역순 학습을 강요받아, 수학에 약한 학생이 과학 자체를 포기하는 구조

**Why now:**
- 브라우저에서 고품질 3D 렌더링과 실시간 물리 시뮬레이션이 실용적 수준에 도달
- XR 기기(Meta Quest, Apple Vision Pro 등) 보급으로 별도 앱 없이 브라우저에서 XR 체험 가능

## 2. Target Users & Use Cases

### Personas

**Primary: "수학이 싫은 민준" (16세, 고1)**
과학은 재밌을 것 같지만 수식이 나오면 흥미를 잃는다. F=ma를 외웠지만 실제로 힘과 가속도의 관계가 어떤 느낌인지 모른다. 직접 물체를 던져보고 힘의 크기를 바꾸며 결과를 눈으로 확인할 수 있다면 수식이 의미하는 바를 체감할 수 있다.

**Secondary: "교양으로 과학을 즐기는 지영" (29세, 개발자)**
물리를 제대로 배운 적 없지만 게임 물리엔진이 어떻게 동작하는지 궁금하다. 에너지 보존, 충돌, 파동 같은 기본 물리를 직관적으로 이해하고 싶다.

**Secondary: "시각적 교보재가 필요한 박 선생님" (35세, 물리 교사)**
수업에서 힘의 벡터, 에너지 변환을 설명할 때 2D 자료의 한계를 느낀다. 학생들이 직접 변수를 조작하며 물리 법칙을 체험할 수 있는 3D 도구가 필요하다.

### Top Use Cases (MVP — 고전물리 트랙)

1. **운동과 힘 체험:** 물체를 던지고, 힘의 크기·방향을 조절하며, 속도/가속도 벡터를 실시간으로 관찰한다
2. **에너지 변환 이해:** 롤러코스터를 직접 설계하고, 운동에너지와 위치에너지 변환을 실시간으로 확인한다
3. **파동 현상 탐구:** 파동을 생성하고, 진폭/파장을 조절하며, 간섭·회절·공명을 시각적으로 체험한다
4. **자유 탐험:** 학습 경로 없이 원하는 시뮬레이션을 선택하고 변수를 자유롭게 조작한다

### Jobs-to-be-Done

> "물리 시간에 F=ma가 나왔을 때, 직접 힘을 바꿔가며 물체가 어떻게 움직이는지 눈으로 보고 싶다. 그래야 수식이 뭘 말하는 건지 와닿는다."

> "롤러코스터를 내 손으로 설계하고, 어디서 속도가 빨라지고 느려지는지 에너지 바로 확인하고 싶다."

## 3. Proposed Solution

브라우저만 열면 바로 3D 물리 시뮬레이션을 체험할 수 있는 웹 플랫폼. XR 기기가 있으면 몰입이 자동으로 강화되는 **3D First, XR Enhanced** 구조.

**핵심 가치:**
1. **만지는 교육** — 슬라이더, 드래그, 토글로 변수를 직접 조작하면 결과가 즉시 변한다
2. **설치 없는 접근성** — 브라우저만 있으면 누구나, 어디서나
3. **직관 우선** — 수식 없이 체험으로 개념을 잡고, 수식은 나중에 연결

**콘텐츠 구조:**
- 4개 트랙: 고전물리(MVP) → 화학 → 우주과학 → 양자역학
- 각 트랙 6개 모듈, 모듈별 3D 인터랙션 + XR 강화 요소
- 상세 커리큘럼: [content/curriculum/](../content/curriculum/)

## 5. Functional Requirements (MVP — Phase 1: 고전물리 트랙)

### 5.1 3D 시뮬레이션 엔진

**[P0]** 사용자가 브라우저에서 3D 물리 시뮬레이션을 60fps로 체험할 수 있다
**[P0]** 마우스/터치로 3D 오브젝트를 회전, 줌, 패닝할 수 있다
**[P0]** 슬라이더, 토글 등 UI 컨트롤로 시뮬레이션 변수를 실시간 조절할 수 있다
**[P0]** 시뮬레이션의 재생, 일시정지, 초기화가 가능하다
**[P1]** XR 지원 기기에서 접속 시 XR 모드로 자동 전환을 제안한다
**[P1]** XR 모드에서 핸드트래킹/컨트롤러로 오브젝트를 조작할 수 있다

### 5.2 모듈 1: 운동과 힘

**[P0]** 물체를 던지거나 밀어서 운동을 만들 수 있다
**[P0]** 힘의 크기와 방향을 슬라이더/드래그로 실시간 조절할 수 있다
**[P0]** 속도/가속도 벡터가 3D 화살표로 오버레이 표시된다
**[P0]** 질량 슬라이더로 물체의 질량을 변경하고 운동 변화를 관찰할 수 있다
**[P1]** 마찰 계수 슬라이더로 마찰력의 영향을 확인할 수 있다
**[P1]** 무중력/지구 중력 환경 토글이 가능하다
**[P1]** 자유낙하 실험: 공기저항 ON/OFF로 낙하 차이를 비교할 수 있다
**[P2]** XR 모드에서 물체를 "손으로" 던지며 힘과 궤적 관계를 체험할 수 있다

### 5.3 모듈 2: 에너지와 일

**[P0]** 롤러코스터 설계 시뮬레이터: 트랙을 직접 그리면 물체가 주행한다
**[P0]** 에너지 바(운동에너지/위치에너지)가 실시간으로 표시된다
**[P0]** 마찰 ON/OFF로 에너지 손실을 관찰할 수 있다
**[P1]** 충돌 시뮬레이션: 질량·속도를 조절하여 탄성/비탄성 충돌을 비교할 수 있다
**[P1]** 진자 운동에서 에너지 변환을 시각화할 수 있다
**[P2]** XR 모드에서 설계한 롤러코스터를 VR로 탑승할 수 있다

### 5.4 모듈 3: 파동

**[P0]** 파동 생성기: 진폭/파장 슬라이더로 파동을 만들고 조절할 수 있다
**[P0]** 두 파동을 합성하여 보강/상쇄 간섭을 관찰할 수 있다
**[P1]** 물결통 시뮬레이션: 이중 파원에서 간섭 패턴을 확인할 수 있다
**[P1]** 공명 시뮬레이션: 진동수를 조절하여 공명점을 찾을 수 있다
**[P2]** XR 모드에서 손으로 파동을 만들어 3D 공간에서 퍼지는 것을 관찰할 수 있다

### 5.5 탐색 & 내비게이션

**[P0]** 트랙 → 모듈 → 인터랙션 구조로 콘텐츠를 탐색할 수 있다
**[P0]** 모듈 내에서 자유 탐험이 가능하다 (선형 학습 경로를 강제하지 않음)
**[P0]** 각 인터랙션에 간결한 설명 텍스트가 제공된다 (수식 없이 직관적 설명)
**[P1]** 모듈 간 관련 개념 링크로 탐색할 수 있다

### 5.6 국제화

**[P0]** UI 및 모든 설명 텍스트가 한국어/영어를 지원한다 (i18n)
**[P0]** 사용자가 언어를 전환할 수 있다
**[P1]** 브라우저 언어 설정에 따라 기본 언어가 자동 설정된다

### 5.7 반응형 & 크로스 플랫폼

**[P0]** 데스크톱 브라우저 (Chrome, Firefox, Edge)에서 동작한다
**[P0]** 모바일 브라우저에서 터치 인터랙션으로 동작한다
**[P1]** XR 지원 브라우저에서 XR 모드가 동작한다
**[P2]** 다크 모드를 지원한다

### 5.8 Telemetry

**[P1]** 모듈별 방문 수, 체류 시간, 인터랙션 횟수를 추적할 수 있다
**[P1]** 사용자 플로우 (어떤 모듈에서 어떤 모듈로 이동하는지)를 추적할 수 있다
**[P2]** B2B 교사 대시보드에서 학생별 학습 진도를 확인할 수 있다

## 6. User Journeys

### Journey 1: 첫 방문 — 자유 탐험

1. 사용자가 PhysPlay 랜딩 페이지에 접속한다
2. 트랙 목록이 표시된다 (Phase 1에서는 고전물리만 활성)
3. 고전물리 트랙을 선택하면 3개 모듈이 표시된다
4. "운동과 힘" 모듈을 선택한다
5. 3D 시뮬레이터가 로드되고 물체와 컨트롤 UI가 표시된다
6. 간결한 안내 텍스트가 인터랙션 방법을 설명한다
7. 사용자가 힘의 크기·방향 슬라이더를 조절한다
8. 물체가 움직이고 속도/가속도 벡터가 실시간 표시된다
9. 질량을 바꿔보며 같은 힘에 다른 가속도가 발생하는 것을 확인한다
10. → Decision: 다음 모듈(에너지와 일)로 이동 / 자유 탐색 계속 / 이탈

### Journey 2: XR 모드 진입

1. 사용자가 XR 지원 기기(Quest 등)의 브라우저로 접속한다
2. 3D 모드로 기본 로드된다
3. "XR 모드로 체험하기" 버튼이 표시된다
4. 버튼을 누르면 XR 세션이 시작된다
5. 동일 콘텐츠가 XR 공간에 배치되고 핸드트래킹/컨트롤러로 조작 가능해진다
6. → Error: XR 미지원 브라우저 → 버튼 미노출, 3D 모드로 정상 동작

### Journey 3: 교사의 수업 활용

1. 교사가 PhysPlay에 접속하여 고전물리 트랙 "에너지와 일" 모듈을 연다
2. 교실 프로젝터/학생 개별 기기로 공유한다
3. 롤러코스터 설계 시뮬레이터에서 트랙을 그리며 에너지 변환을 설명한다
4. 학생들이 각자 기기에서 자신만의 롤러코스터를 설계한다
5. → Future (B2B): 교사가 대시보드에서 학생 참여도를 확인한다

## 7. Scope & Non-Goals

### In Scope (Phase 1)
- 고전물리 트랙 3개 모듈 (운동과 힘, 에너지와 일, 파동)
- 3D 물리 시뮬레이션 엔진
- XR 모드 지원 (3D → XR 전환)
- 한국어/영어 i18n
- 반응형 웹 (데스크톱 + 모바일)
- 랜딩 페이지

### Out of Scope
- 화학, 우주과학, 양자역학 트랙 (Phase 2-4)
- 사용자 계정/로그인 시스템 (Phase 1은 익명 접근)
- B2B 교사 대시보드 (Phase 1에서는 파일럿 확보만, 대시보드는 Phase 2)
- 네이티브 앱 (웹 전용)
- 학습 진도 저장/추적 (계정 시스템 이후)
- 교과 매핑/시험 문제 연동 (자유 체험 중심)
- 소셜 기능 (댓글, 공유, 멀티플레이어)

### Future Consideration
- 화학 트랙: PubChem/PDB 연동 분자 조작
- 양자역학 트랙의 이중슬릿 실험 데모를 바이럴 마케팅 소재로 활용
- 고전물리 트랙 확장: 소리와 빛, 전기와 자기, 전자기파 모듈
- B2B SaaS: 교사 대시보드, 학습 진도 추적, 기관 라이선스
- 콘텐츠 에디터: 교사가 커스텀 시뮬레이션 시나리오를 생성

## 8. Assumptions, Constraints & Dependencies

### Assumptions
- 중고등학생은 3D 인터랙션이 2D 다이어그램보다 물리 개념 이해에 효과적이다
- Mid-range 디바이스(2020년 이후 출시)에서 브라우저 기반 60fps 물리 시뮬레이션이 가능하다
- 롤러코스터 설계 같은 게이미피케이션 요소가 학습 참여도를 높인다

### Constraints
- 웹 전용 — 네이티브 앱 없음
- 한국어/영어 2개 언어만 지원
- Phase 1에서 계정 시스템 없음 (서버 비용 최소화)
- XR 모드는 일부 브라우저에서만 안정적 지원 (Safari 제한적)

### External Data Sources

| Phase | 트랙 | 데이터 소스 | 용도 |
|-------|------|------------|------|
| Phase 1 | 고전물리 | 자체 생성 | 기본 3D 지오메트리, 물리 오브젝트 |
| Phase 2 | 화학 | PubChem, PDB (Protein Data Bank) | 분자 구조, 화합물 정보, 단백질 3D 구조 |
| Phase 3 | 우주과학 | NASA Open API, ESA | 천체 데이터, 텍스처, 궤도 정보 |
| Phase 4 | 양자역학 | 자체 생성 | 확률 분포, 파동함수 시각화 |

### Risks

| Risk | Severity | Mitigation |
|------|----------|------------|
| 물리 시뮬레이션이 모바일에서 성능 부족 | High | 시뮬레이션 복잡도 제한, 모바일용 경량 모드 |
| XR 브라우저 지원 파편화 | Medium | 3D First 철학 — XR은 보너스, 3D가 기본 |
| PhET 등 무료 대안 대비 차별화 부족 | Medium | 3D-native + 물리엔진 기반 리얼타임 시뮬레이션 + 게이미피케이션 |
| B2B 학교 의사결정 사이클이 길어 파일럿 확보 지연 | Medium | 무료 접근 + 교사 개인 단위 접근 병행 |

## 1. Problem / Opportunity

### The Problem

과학 교육에는 세 겹의 문제가 겹쳐 있다.

**(1) 차원의 문제 -- 3D 개념을 2D 매체로 전달하는 한계.**
분자 구조, 전자 오비탈, 파동 간섭, 궤도 역학 등 본질적으로 3차원인 개념이 2D 교과서와 정적 다이어그램으로 전달된다. 학생은 머릿속에서 3D를 재구성해야 하며, 이 인지 부담이 이해를 가로막는다.

**(2) 순서의 문제 -- 수식이 직관보다 먼저 온다.**
수식이 직관보다 먼저 제시되는 역순 학습 구조에서, 수학에 약한 학생은 과학 자체를 포기한다. "F=ma를 외웠지만 힘과 가속도의 관계가 어떤 느낌인지 모른다"는 것이 전형적인 증상이다.

**(3) 수동성의 문제 -- "만질 수 있다"와 "배운다"는 다르다.**
PhET처럼 인터랙티브 시뮬레이션이 있어도, 슬라이더를 만지작거리다 나가는 것이 현실이다. 능동적 사고를 유도하는 구조가 없으면 "만지는 교육"도 "보는 교육"과 다르지 않다. 학습 과학 연구에 따르면, 진짜 개념 변화(conceptual change)는 **"내 예측이 틀렸다"는 인지적 갈등(cognitive conflict)** 순간에 발생한다. 현재 어떤 도구도 이 순간을 체계적으로 설계하지 않는다.

### Why Now

WebGPU 표준화와 Three.js/R3F 생태계 성숙으로 브라우저에서 실시간 3D 물리 시뮬레이션이 실용적 수준에 도달했고, WebXR로 동일 코드베이스에서 XR 전환이 가능하다. Duolingo가 언어 교육을 재정의한 것처럼 3D 체험과 결합한 과학 교육 게이미피케이션은 아직 없다.

### Evidence Summary

- OECD PISA 2022 기준, 한국 학생의 과학 흥미도는 OECD 평균 이하. 과학 진로 선택 비율은 지속 하락 추세
- 기존 도구 한계: 2D 위주(PhET), 고가 설치형($50K+/학교, Labster), 수동적 시청(YouTube), 문제풀이 중심(Brilliant, Khan Academy)
- 3D 체험 + 능동적 사고 유도 + 학습 동기 엔진을 통합한 도구가 존재하지 않음

| 현재 도구 | 강점 | 한계 | 빠진 것 |
|-----------|------|------|---------|
| 교과서 | 체계적 커리큘럼 | 2D 정적, 수식 우선 | 체험, 동기 |
| YouTube | 뛰어난 시각화, 무료 | 수동적 시청 | 능동적 사고, 피드백 |
| PhET Simulations | 인터랙티브, 무료 | 대부분 2D, 목표 없는 탐색 | 학습 동기, 구조화된 경험 |
| Brilliant.org | 인터랙티브 문제풀이 | 3D 제한적, 문제풀이 중심 | 3D 체험, 실험적 발견 |
| Labster | VR 실험실 | 고가($50K+/학교), 설치형 | 접근성, 적응형 경험 |

## 2. Target Users & Use Cases

### Personas

**서연 (10세, 초4) -- "과학이 지루한 초등학생"**
과학 시간에 배운 개념(힘과 운동, 물질의 상태)을 직접 만져보고 싶지만, 교과서는 글자만 많고 재미없어서 곧 포기한다. "내가 직접 만져보고, 맞추고, 틀리면 왜 틀렸는지 바로 볼 수 있으면 과학 시간이 게임처럼 재밌을 것 같다."

**민준 (16세, 고1) -- "수학이 싫은 고등학생"**
물리 수업에서 투사체 운동 공식을 외웠지만, 실제로 어떤 느낌인지 직관적으로 이해하지 못한다. PhET에서 슬라이더를 조작해봤지만 목표 없는 탐색이라 곧 이탈한다. "내가 예측하고, 실험하고, 내 생각이 맞는지 틀렸는지 바로 확인할 수 있으면 -- 다음 단계가 궁금해서 계속하게 될 것 같다."

**지영 (29세, 개발자) -- "커리어 전환 중인 성인 학습자"**
양자컴퓨팅에 관심이 생겨 양자역학 기초를 잡고 싶지만, 교과서와 온라인 강의는 수식 장벽이 높고 자기 수준에 맞지 않는다. "틀려도 괜찮은 환경에서 직접 해보면서 양자역학 직관을 잡고 싶다."

## 3. Proposed Solution

### Elevator Pitch

과학 개념을 **"예측하고, 실험하고, 발견하는" 3D 게임**으로 만든다. 사용자는 매 챌린지에서 결과를 먼저 예측하고, 3D 시뮬레이션에서 God Hand(1인칭 테이블탑 조작)로 직접 실험한 뒤, 예측과 결과를 비교하며 과학 원리를 스스로 발견한다. 시뮬레이션 엔진 기반의 무한 챌린지 생성과 AI 적응형 난이도 조절로, 틀린 예측이 다음 단계의 호기심이 되는 자기강화 루프를 만든다.

### Core Value Propositions

| Value Prop | 해결하는 문제 (Section 1) |
|-----------|-------------------------|
| **예측-실험-발견 코어 루프** | 수동성 -- "생각하지 않고 만지는" 기존 도구와 달리, 인지적 갈등을 핵심 메카닉으로 설계 |
| **3D-native 시뮬레이션** | 차원 -- 3D 개념을 3D로 직접 보고, 만지고, 실험 |
| **직관 우선, 수식은 나중** | 순서 -- 먼저 체험으로 직관을 잡고, 수식은 이해의 도구로 나중에 등장 |

### Mental Model

**"나만의 과학 연구소"** -- 사용자는 자신의 연구소에서 실험자로서 과학을 탐험한다. 연구소에는 여러 공간(역학 실험실, 분자 실험실, 우주 관측소, 양자 연구소)이 있고, 각 공간에는 여러 스테이션(실험대)이 있다. 스테이션에서는 무한한 챌린지가 생성되며, 챌린지를 클리어하면 새로운 스테이션과 공간이 해금된다. 게임을 할수록 내 연구소가 넓어지고 깊어진다.

## 5. Functional Requirements

### Core Loop

- [REQ-001] 사용자는 챌린지 진입 시 실험 결과를 먼저 예측할 수 있다. 예측 방식은 챌린지 성격에 따라 궤적 그리기(trajectory), 이진 선택(binary), 패턴 선택지(pattern), 3D 배치(placement) 중 하나로 결정된다
- [REQ-002] 사용자는 예측 제출 후 3D 시뮬레이션에서 God Hand(1인칭 테이블탑 조작)로 직접 실험을 수행할 수 있다
- [REQ-003] 실험 완료 후 사용자의 예측과 실제 결과가 시각적으로 비교되며(Discover 단계), 관련 과학 개념 설명이 제공된다
- [REQ-004] Discover 단계의 개념 설명은 3단계 깊이를 가진다: Level 1(직관적 비유), Level 2(개념 설명), Level 3(수식 포함). 사용자 수준에 따라 적절한 깊이가 선택된다
- [REQ-005] 챌린지 완료 후 사용자 수준에 맞는 다음 챌린지가 제시된다

### Prediction Types

- [REQ-006] 궤적 그리기(trajectory): 사용자가 화면에 궤적을 직접 그리고, 실제 궤적과 오차 범위 기반으로 정답이 판정된다
- [REQ-007] 이진 선택(binary): "될까 / 안 될까", "이쪽 / 저쪽" 같은 양자택일로 정답/오답이 판정된다
- [REQ-008] 패턴 선택지(pattern): 여러 결과 패턴 중 하나를 고르고, 선택지 매칭으로 판정된다
- [REQ-009] 3D 배치(placement): 3D 공간에 마커/오브젝트를 직접 배치하고, 위치/방향 오차 기반으로 판정된다

### God Hand Interaction

- [REQ-010] 사용자는 캐릭터/아바타 없이 1인칭 실험자로서 테이블탑 위의 대상을 직접 조작한다
- [REQ-011] God Hand 인터랙션은 던지기/발사, 조립/배치, 당기기/밀기, 설치/제거, 공간 흔들기 패턴의 조합으로 구성된다
- [REQ-012] PC에서는 마우스/키보드로 조작한다 (클릭 드래그로 던지기, 마우스 드로잉, 스크롤 줌, 키보드 단축키)
- [REQ-013] 모바일에서는 터치 제스처로 핵심 경험이 유지된다 (스와이프로 던지기, 손가락 드로잉, 핀치 줌)
- [REQ-014] XR에서는 핸드 트래킹으로 동일한 God Hand 멘탈 모델이 유지된다 (직접 손으로 집기/던지기, 공중 드로잉)

### Screen Composition

- [REQ-015] 실험실 진입 시 화면 전체가 3D 뷰포트이며, 모든 UI(예측 입력, 개념 설명, 버튼, 변수 조절)는 3D 위에 떠 있는 HUD 오버레이로 표시된다
- [REQ-016] 연구소 허브(공간/스테이션 선택), 진도, 설정 등 실험실 바깥 화면은 2D 페이지로 처리된다

### Content Architecture: Engine-based Structure

- [REQ-017] 시뮬레이션 엔진이 물리 법칙을 실시간으로 계산하고 3D로 렌더링한다. 각 엔진은 독립적이며, 하나의 엔진 위에서 다양한 챌린지가 생성된다
- [REQ-018] 챌린지는 "엔진 + 변수 조합(params) + 예측 질문 + 메타데이터(tags, difficulty, relatedConcepts)"로 구성된 JSON 데이터로 정의된다
- [REQ-019] 챌린지 데이터에 predict.type으로 예측 방식이 정의되어, UGC에서도 유저가 예측 방식을 선택할 수 있다
- [REQ-020] Discover 콘텐츠는 concept-level 라이브러리로 구조화된다. 개념별 설명을 한 번만 작성하고, 챌린지가 relatedConcepts로 해당 개념을 참조한다
- [REQ-021] 정답/오답에 따른 프레이밍(축하, 오류 설명)은 챌린지 JSON에 하드코딩하지 않고, 사용자의 정답/오답 상태 + concept 깊이 레벨에 맞춰 런타임 로직으로 조합된다

### Simulation Engines

- [REQ-022] 투사체 엔진: 포물선 운동, 중력, 마그누스, 코리올리, 중력보조, 시공간 곡률
- [REQ-023] 충돌/에너지 엔진: 마블런, 충돌, 에너지 보존, 진자, 루브 골드버그, 푸코 진자
- [REQ-024] 파동 엔진: 정상파, 간섭, 도플러, 마하콘, 공명, 건축음향, 악기물리
- [REQ-025] 분자 엔진: VSEPR, 입체화학, 극성, 반응속도론, 단백질접힘, DNA, 결정구조
- [REQ-026] 궤도 엔진: 케플러, 조석력, 라그랑주, 삼체문제, 쌍성계, 은하충돌
- [REQ-027] 양자 엔진: 이중슬릿, 터널링, 오비탈, 불확정성, 얽힘, 광전효과

### Cross-Reference Knowledge Graph

- [REQ-028] 엔진 간에 공유되는 물리 개념을 크로스 참조 맵으로 연결한다. 사용자가 한 엔진에서 개념을 학습한 후 다른 엔진의 관련 챌린지를 추천하는 크로스 엔진 추천의 데이터 기반이 된다

크로스 참조 예시:

| 개념 | 주 엔진 (정의) | 참조 엔진 (사용) | 연결 맥락 |
|------|---------------|-----------------|----------|
| 중력 | 투사체 (표면 중력) | 궤도 (만유인력) | "지구에서의 g=9.8은 만유인력의 특수한 경우" |
| 적색편이 | 파동 (도플러 효과) | 궤도 (은하 후퇴 속도) | "파동에서 배운 도플러가 우주 팽창의 증거가 된다" |
| 전자기 스펙트럼 | 파동 (파장/진동수) | 양자 (광전효과, 흑체복사) | "파동의 에너지가 양자화되어 있다" |
| 에너지 양자화 | 양자 (플랑크) | 분자 (전자 에너지 준위) | "원자의 전자가 불연속 에너지 준위에 있는 이유" |
| 파동 간섭 | 파동 (물결, 소리) | 양자 (이중슬릿) | "물결 간섭과 같은 원리가 양자 세계에서도 작동" |
| 확률구름/오비탈 | 양자 (파동함수) | 분자 (전자 배치, 결합) | "오비탈의 모양이 화학결합의 방향을 결정" |

크로스 참조 맵의 상세 정의는 `docs/contents/engines/README.md`를 원본으로 한다.

### Spaces -- Game Stages

- [REQ-029] 4개 공간이 게임 스테이지로 구성된다. 각 공간은 고유한 비주얼 테마(Skybox, 조명, 색상 팔레트, 머티리얼, 파티클 이펙트), 앰비언트 사운드, BGM, 전환 연출을 가진다

| 공간 | 트랙 | 포함 엔진 |
|------|------|----------|
| 역학 실험실 | 고전물리 | 투사체, 충돌/에너지, 파동 |
| 분자 실험실 | 화학 | 분자 |
| 우주 관측소 | 우주과학 | 궤도 |
| 양자 연구소 | 양자역학 | 양자 |

- [REQ-030] 실험실을 해금하는 것 자체가 "새로운 스테이지를 여는" 경험을 제공한다. 실험실 진입 시 전환 연출, 고유한 BGM, 시각적 리워드 이펙트가 동반된다

### Content Expansion Model

- [REQ-031] 콘텐츠는 세 방향으로 확장 가능하다: 가로 확장(새 공간), 세로 확장(기존 공간에 새 스테이션), 깊이 확장(기존 엔진에 챌린지 추가)

### AI System: Adaptive Experience Engine

- [REQ-032] 챌린지 풀에서 메타데이터(tags, difficulty, relatedConcepts)를 기반으로 사용자 수준에 맞는 다음 챌린지를 동적으로 선택한다
- [REQ-033] 연속 정답/오답에 따라 difficulty 필터 범위를 조정하여 적절한 난이도의 챌린지를 선택한다
- [REQ-034] tags별 정확도를 추적하여 낮은 정확도 태그의 챌린지를 우선 배치한다 (약점 보강)
- [REQ-035] Discover 단계에서 사용자 수준에 맞는 concept-level 설명 깊이(level1/2/3)를 선택한다

AI Phase별 구현 전략:

| Phase | 접근 방식 | 설명 |
|-------|----------|------|
| Phase 1 | 메타데이터 기반 규칙 엔진 | 챌린지의 tags/difficulty/relatedConcepts 메타데이터 위에서 규칙 기반으로 다음 챌린지를 선택. 메타데이터 필터링 + 우선순위 규칙 |
| Phase 2+ | ML 모델 전환 | Phase 1에서 쌓인 메타데이터 + 사용자 행동 데이터로 학습, 개인화 정확도 향상 |

### Onboarding

- [REQ-036] 첫 방문 시 회원가입/트랙 선택 없이 바로 역학 실험실의 "공 던지기" 챌린지로 진입한다. 30초 안에 예측 -> 실험 -> 발견 한 바퀴를 체험한다
- [REQ-037] 첫 챌린지 완료 후 연구소 허브가 열리며 전체 공간 맵이 등장한다. 해금된 공간과 잠긴 공간(실루엣)이 보여 호기심을 유발한다
- [REQ-038] 재방문 시 연구소 허브(2D)가 홈 화면이 된다. 공간 선택 시 전환 연출과 함께 풀스크린 3D + HUD 실험실로 진입한다

### Visual Design Direction

- [REQ-039] 2D 레이어(Landing, Hub)는 Soft-Tech Educational 톤으로, 밝고 가벼운 색감과 부드러운 라운드로 교육 플랫폼 신뢰감을 제공한다
- [REQ-040] 2D에서 3D로의 전환(Portal Transition)은 극적인 연출로 몰입 전환의 심리적 게이트 역할을 한다
- [REQ-041] 3D 실험실은 Semi-Stylized 3D Sandbox 톤으로, 교육 도구가 아니라 게임 스테이지처럼 느껴지는 것이 핵심이다

상세 비주얼 디자인 언어(컬러, 타이포, 모션 원칙 등)는 UX Design 문서를 참조한다.

### XR Strategy

- [REQ-042] 모든 챌린지는 3D에서만 의미 있어야 하며(Must 3D), XR이 체험을 질적으로 증폭해야 한다(Must XR). 2D에서는 불가능하거나 무의미한 챌린지만 만든다
- [REQ-043] XR 없이도 100% 동작하되, XR 전환 시 동일 씬에서 입력만 핸드 트래킹으로 전환된다

XR 코어 루프별 증폭 효과:

| 코어 루프 단계 | XR 증폭 효과 |
|-------------|------------|
| Predict | 실제로 손으로 궤적을 그리며 예측. 신체적 예측이 인지적 참여를 높인다 |
| Play | 공간에서 직접 던지고, 분자를 손으로 조립. "내 손의 감각"이 개념 이해에 연결된다 |
| Discover | 예측한 궤적과 실제 궤적이 3D 공간에서 겹쳐 보임. 틀린 지점이 공간적으로 드러난다 |
| 환경 전환 | 방 전체의 환경(조명, 배경, 사운드스케이프)이 실험실 테마에 맞게 바뀌며 게임 스테이지 몰입감을 극대화 |

Progressive XR 전략:
- Phase 1~: 모든 챌린지 설계에 Must 3D/XR + XR 증폭 시나리오 포함
- Phase 1 구현: 3D 웹. XR 대응 아키텍처 확보 (God Hand 멘탈 모델 + 동일 코드베이스)
- Phase 2+: XR 모드 구현. 동일 씬에서 입력만 핸드 트래킹으로 전환

### UGC (User-Generated Content)

- [REQ-044] 사용자가 엔진의 변수를 세팅하고 챌린지를 제작하여 URL로 공유할 수 있다 (Phase 3)
- [REQ-045] 사용자가 챌린지를 엮어 스테이션/미션 시퀀스를 제작할 수 있다 (Phase 4+)
- [REQ-046] 사용자가 테마 프리셋을 조합하고 스테이션을 배치하여 공간을 제작할 수 있다 (Phase 5+)
- [REQ-047] Phase 1부터 챌린지/스테이션/공간을 코드가 아닌 데이터(JSON)로 정의한다. 엔진은 이 데이터를 읽어서 실행하는 구조로, 나중에 에디터 UI만 붙이면 UGC가 열린다

### Analytics

- [REQ-048] 코어 루프 퍼널(Predict -> Play -> Discover) 단계별 이탈률을 추적할 수 있다
- [REQ-049] 예측 참여율 및 스킵율을 모니터링할 수 있다
- [REQ-050] 스테이션 내 예측 정확도 변화를 시계열로 추적할 수 있다 (학습 효과)
- [REQ-051] 스테이션/공간 간 이동 흐름을 파악할 수 있다 (탐험 패턴)

### Non-functional

## 6. User Journeys

### Journey 1: First Visit -- 첫 30초 체험

1. 사용자가 URL을 열면 회원가입/트랙 선택 없이 바로 역학 실험실의 "공 던지기" 챌린지로 진입한다
2. **[Predict]** "이 공을 던지면 어디에 떨어질까?" -- 사용자가 궤적을 그린다
   - 예측을 건너뛰려는 경우 -> 가벼운 유도 메시지 후 스킵 허용
3. **[Play]** God Hand로 공을 클릭 드래그하여 직접 던진다
4. **[Discover]** 예측한 궤적과 실제 궤적이 겹쳐 표시된다
   - 틀렸다면 -> "왜 다를까?" + Level 1 직관적 비유 설명 + 변수 조절 유도
   - 맞았다면 -> 축하 + 변수 추가(다른 중력 환경 등)로 더 어려운 예측 유도
5. 첫 챌린지 완료 후 연구소 허브가 열린다
6. 전체 공간 맵이 등장 -- 해금된 공간과 잠긴 공간(실루엣)이 보여 호기심을 유발한다

**Drop-off risk:** Step 2(예측 단계). 처음부터 예측을 요구하면 부담스러울 수 있다. 첫 챌린지의 예측은 매우 직관적이고 저부담이어야 한다.

### Journey 2: Returning User -- 스테이션 탐색

1. 사용자가 연구소 허브(2D 홈 화면)에 진입한다
2. 해금된 공간들이 보이고, 잠긴 공간은 실루엣으로 표시된다
3. 사용자가 역학 실험실을 선택한다
4. 전환 연출(Portal Transition)과 함께 풀스크린 3D + HUD 실험실로 진입한다
5. HUD에서 스테이션(투사체/에너지/파동) 중 하나를 선택한다 -- 3D를 벗어나지 않음
6. 이전 진도에 이어서 코어 루프(Predict -> Play -> Discover) 반복
7. 챌린지 완료 시 적응형 엔진이 메타데이터 기반으로 다음 챌린지를 선택한다
   - 연속 정답 -> difficulty 상승
   - 연속 오답 -> difficulty 하락 + 약점 태그 우선 배치
8. 스테이션 완주 시 새로운 스테이션 또는 공간이 해금된다

**Drop-off risk:** Step 6(반복 챌린지). 같은 유형의 챌린지가 반복되면 지루함을 느낄 수 있다. 변수 전환 규칙으로 다양성을 확보해야 한다.

### Journey 3: Cross-Engine Discovery -- 개념 연결

1. 사용자가 파동 스테이션에서 도플러 효과 챌린지를 완료한다
2. Discover 단계에서 "적색편이" 개념이 설명된다
3. 크로스 참조 추천: "이 개념이 우주에서는 어떻게 적용될까? -- 우주 관측소에서 확인해보세요"
4. 사용자가 추천을 따라 우주 관측소의 관련 챌린지로 이동한다
   - 우주 관측소가 아직 잠겨 있는 경우 -> 해금 조건을 안내하고 호기심을 유발한다
5. 다른 엔진에서 같은 개념이 다른 맥락으로 적용되는 것을 체험한다

## 7. Scope & Non-Goals

### In Scope (Full Vision)

- 예측-실험-발견 코어 루프 + God Hand 1인칭 테이블탑 조작
- 6개 시뮬레이션 엔진 (투사체, 충돌/에너지, 파동, 분자, 궤도, 양자)
- 4개 공간 (역학 실험실, 분자 실험실, 우주 관측소, 양자 연구소)
- 엔진 기반 무한 챌린지 생성 + AI 적응형 경험 엔진
- 풀스크린 3D + HUD 화면 구성
- Progressive XR (3D 웹 -> XR 모드)
- UGC (챌린지 -> 스테이션 -> 공간 제작)
- 크로스 엔진 지식 그래프 및 추천
- en/ko 다국어 지원

### Out of Scope

| Non-Goal | 근거 |
|----------|------|
| B2B / 학교 기관 판매, 교사 대시보드 | B2C 개인 사용자 경험에 집중. 기관 판매는 코어 경험 검증 후 별도 검토 |
| 공인 교육과정/학점 연동 | 체험과 이해가 목적. 교육과정 인증은 다른 종류의 제품 |
| 네이티브 앱 | 웹 기반, URL 하나로 접근. 모바일은 반응형 웹으로 지원 |
| AI 튜터/챗봇 | AI는 풀 챗봇이 아니라 적응형 경험 엔진(디렉터 AI)으로 작동 |
| 3인칭 RPG / 아바타 | 1인칭 실험자 + God Hand. 캐릭터 개발 복잡도 완전 제거 |
| 24개 모듈 수동 제작 | 엔진 기반 무한 챌린지 생성. 콘텐츠 확장은 엔진 + 변수 조합으로 해결 |
| Phase 1에서 계정 시스템 | 로컬 저장으로 시작. 계정은 Phase 3 |
| Phase 1에서 XR 모드 구현 | Must 3D/XR 콘텐츠 설계 + XR 대응 아키텍처는 확보하되, 실제 XR 구현은 Phase 2+ |

**What this document does NOT cover:**
- UI/UX 디자인, 와이어프레임, 인터랙션 패턴, 비주얼 스펙 -> UX Design 문서
- 기술 아키텍처, 기술 스택, API 설계 -> Design Doc, Client Structure
- 스프린트 계획, 태스크 분해 -> 별도 프로젝트 관리

## 8. Assumptions, Constraints & Risks

### Assumptions

| # | Assumption | Impact | Validation Plan |
|---|-----------|--------|----------------|
| A1 | "예측 -> 실험 -> 발견" 루프가 자유 탐색보다 학습 효과가 높다 | High | A/B 테스트: 자유 탐색 vs 예측 루프 모드, 사전/사후 이해도 비교 (n=20) |
| A2 | 초등학생부터 성인까지 동일 코어 루프로 커버할 수 있다 | High | 연령대별 사용자 테스트 -- 예측 참여율/완주율/재방문율 비교 |
| A3 | 예측 참여가 학습 동기(다음 모듈 진행)를 실제로 높인다 | High | Phase 1에서 모듈 완주율 측정 (목표: 40%) |
| A4 | 3D 인터랙션이 2D 다이어그램보다 개념 이해도를 높인다 | High | 사용자 테스트에서 사전/사후 이해도 비교 |
| A5 | 엔진 기반 변수 조합 챌린지가 수동 제작 챌린지만큼 교육 효과가 있다 | Medium | Phase 1에서 수동 설계 챌린지 vs 변수 조합 챌린지 비교 |

### Constraints

| # | Constraint |
|---|-----------|

## 9. Timeline & Milestones

### Wedge Strategy

역학 실험실(고전물리)부터 시작하는 이유:
1. 최대 사용자 풀 -- 초등부터 성인까지 모든 연령이 경험하는 가장 기본적인 과학 영역
2. 직관적 인터랙션 -- 투사체, 롤러코스터는 게이미피케이션에 최적화
3. 코어 루프 최적 적용 -- "예측 -> 실험 -> 발견" 루프를 가장 자연스럽게 적용 가능
4. God Hand 검증 -- 핵심 인터랙션(집기, 던지기, 그리기)을 검증하기에 최적
5. 풍부한 변수 공간 -- 엔진 3개(투사체, 충돌/에너지, 파동)로 다양한 챌린지 확보
6. 외부 의존 없음 -- 외부 데이터 없이 빠른 출시 가능
7. 기술 기반 검증 -- 엔진 아키텍처, 코어 루프, 적응형 분기 등 공통 기술 기반 검증

양자 연구소가 마지막인 이유: 이중슬릿 실험의 "관측기 ON/OFF"는 바이럴 잠재력과 게이미피케이션 적합도가 동시에 가장 높다. 역학 실험실로 기술과 코어 루프를 검증한 뒤, 양자 연구소를 바이럴 + 게임 경험의 정점으로 활용한다.

### Phased Roadmap

| Phase | Duration | Key Activities | 확장 방향 |
|-------|----------|---------------|----------|
| Discovery | 2-3주 | 투사체 엔진 프로토타입 + God Hand + 코어 루프 통합 | -- |
| Phase 1 (역학 실험실) | 2-3개월 | 엔진 3개(투사체, 충돌/에너지, 파동) + 역학 실험실 공간 + 규칙 기반 적응형 분기 + 챌린지 생성 시스템 + 랜딩 + i18n + 트래킹 + 출시 | 첫 공간 |
| Phase 2 (역학 확장 + AI) | 1-2개월 | 역학 실험실에 소리/빛, 전자기 스테이션 추가 + 적응형 AI v1 + 피드백 반영 | 세로 확장 |
| Phase 3 (분자 실험실) | 3-4개월 | 분자 엔진 + 분자 실험실 공간 + 계정/진도 시스템 + 챌린지 에디터(UGC 1단계) | 가로 확장 |
| Phase 4 (우주 관측소) | 3-4개월 | 궤도 엔진 + 우주 관측소 공간 + NASA API 연동 + 스테이션/미션 제작(UGC 2단계) | 가로 확장 |
| Phase 5 (양자 연구소) | 3-4개월 | 양자 엔진 + 양자 연구소 공간 + 바이럴 마케팅 + 공간 제작(UGC 3단계) | 가로 확장 |
| Phase 6+ | Ongoing | 기존 공간 세로 확장 + 깊이 확장 + 신규 공간 (생명과학, 유체역학 등) | 모든 방향 |

### Phase Expansion Tree

```
Phase 1 (Launch)
+-- 역학 실험실
    +-- 투사체 스테이션 (투사체 엔진)
    +-- 에너지 스테이션 (충돌/에너지 엔진)
    +-- 파동 스테이션 (파동 엔진)

Phase 2
+-- 역학 실험실
    +-- 투사체 스테이션
    +-- 에너지 스테이션
    +-- 파동 스테이션
    +-- 소리/빛 스테이션 <-- 세로 확장 (파동 엔진 재사용 + 새 변수)
    +-- 전자기 스테이션 <-- 세로 확장 (새 엔진)

Phase 3
+-- 역학 실험실 (계속 확장 가능)
+-- 분자 실험실 <-- 가로 확장 (새 공간 + 분자 엔진)
    +-- 원자 결합 스테이션
    +-- 분자 구조 스테이션
    +-- 화학 반응 스테이션

Phase 4
+-- 역학 실험실
+-- 분자 실험실
+-- 우주 관측소 <-- 가로 확장 (새 공간 + 궤도 엔진)

Phase 5
+-- 역학 실험실
+-- 분자 실험실
+-- 우주 관측소
+-- 양자 연구소 <-- 가로 확장 (새 공간 + 양자 엔진)
```

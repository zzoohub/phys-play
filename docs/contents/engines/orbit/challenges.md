# 궤도 엔진 — 챌린지 시나리오 v2

> 소스: 3-1. 지구와 달 + 3-2. 태양계 + 3-3. 별의 일생 + 3-4. 빛과 스펙트럼 + 3-5. 은하
> v2 원칙: Must 3D/XR · 게이미피케이션 핵심 · WOW Point 필수

## v2 설계 철학

| 원칙 | 적용 |
|------|------|
| Must 3D | 궤도는 3D 공간의 현상. 궤도면 기울기, 세차운동, 3체 문제는 2D로 불가 |
| WOW Point | "내가 만든 궤도가 카오스로 붕괴한다?" — 우주의 역학적 아름다움과 불안정성 |
| God Hand | 행성을 손으로 밀어 궤도 투입, 천체를 잡아당겨 조석력 체험, 시간 다이얼 조작 |
| XR 증폭 | 태양계 한가운데 서서 행성이 머리 위를 지나감. 규모를 체감하는 축소/확대 |

## 시나리오 풀

### #1 태양계 건축가 (Solar System Architect)

| 항목 | 내용 |
|------|------|
| **컨셉** | 빈 공간에 항성과 행성을 배치하고 초기 속도를 줘서 안정한 궤도를 만든다 |
| **Must 3D** | 궤도면 기울기, 이심률에 따른 3D 타원 궤도. 2D에서는 기울어진 궤도 표현 불가 |
| **변수 세팅** | `star={mass:1.989e30}`, `planets=[]`, `mode=sandbox` |
| **예측 (placement)** | "이 행성에 어떤 방향/속도로 밀어야 안정한 원궤도가 될까?" |
| **God Hand** | 행성을 손으로 잡아 던진다 — 던지는 방향과 세기가 궤도를 결정 |
| **WOW Point** | 속도가 조금만 빨라도 타원 → 포물선으로 튕겨나간다. "이게 탈출 속도!" |
| **XR 증폭** | 태양 옆에 서서 행성이 머리 위를 지나가는 것을 올려다본다. 규모감 |
| **Discover 핵심** | 케플러 법칙, 궤도 속도, 탈출 속도, 원궤도 vs 타원궤도 |
| **난이도** | 2 |

### #2 일식 엔지니어 (Eclipse Engineer)

| 항목 | 내용 |
|------|------|
| **컨셉** | 태양-지구-달의 3D 배치를 조정하여 개기일식을 만든다 |
| **Must 3D** | 달의 궤도면이 5.14° 기울어짐 — 이 3D 기울기 때문에 일식이 매번 안 일어남 |
| **변수 세팅** | `bodies=[sun,earth,moon]`, `moonOrbitTilt=5.14`, `moonPhase=variable`, `timeScale=fast` |
| **예측 (placement)** | "달을 어디에 놓아야 지구에서 개기일식을 볼 수 있을까?" |
| **God Hand** | 달의 궤도면 기울기를 손으로 조절. 달을 궤도 위에서 이동시켜 정렬 시도 |
| **WOW Point** | 달의 시직경과 태양의 시직경이 거의 같다는 우주적 우연 — 지구에서 볼 때 "딱" 가린다 |
| **XR 증폭** | 지구 표면에 서서 달 그림자가 다가오는 것을 목격. 코로나가 보이는 순간 |
| **Discover 핵심** | 궤도면 기울기, 교점(lunar node), 시직경, 일식의 조건 |
| **난이도** | 3 |

### #3 조석력 실험실 (Tidal Force Lab)

| 항목 | 내용 |
|------|------|
| **컨셉** | 달의 위치/거리를 조절하여 지구의 조석 변형을 관찰하고 예측 |
| **Must 3D** | 조석력은 3D 기조력(tidal force) — 지구 양쪽으로 늘어나는 럭비공 형태 |
| **변수 세팅** | `earth={radius:6371km}`, `moonDistance=variable`, `showTidalBulge=true`, `showForceField=true` |
| **예측 (pattern)** | "달이 더 가까워지면 조석 팽출(tidal bulge)은 어떻게 변할까?" |
| **God Hand** | 달을 손으로 잡아 지구에 가까이/멀리 이동. 실시간으로 지구의 변형이 보인다 |
| **WOW Point** | 로슈 한계(Roche limit) — 달을 너무 가까이 가져가면 조석력이 달 자체를 부순다! → 토성의 고리 형성 |
| **XR 증폭** | 지구 해변에 서서 달을 가까이 가져갈수록 해수면이 올라오는 것을 체감 |
| **Discover 핵심** | 기조력, 조석 팽출, 로슈 한계, 동주기 자전 |
| **난이도** | 3 |

### #4 라그랑주 포인트 헌트 (Lagrange Point Hunt)

| 항목 | 내용 |
|------|------|
| **컨셉** | 태양-지구 시스템에서 중력이 균형을 이루는 5개의 라그랑주 점을 찾는다 |
| **Must 3D** | L4, L5는 궤도면 위의 정삼각형 꼭짓점 — 3D 기하학 없이는 위치를 이해 불가 |
| **변수 세팅** | `bodies=[sun,earth]`, `probeCount=5`, `showGravityPotential=true`, `showRotatingFrame=true` |
| **예측 (placement)** | "탐사선이 영원히 같은 상대 위치에 머물 수 있는 지점 5개를 찾아봐" |
| **God Hand** | 작은 탐사선을 3D 공간의 여러 지점에 놓아본다. 유효 중력 포텐셜이 컬러맵으로 표시 |
| **WOW Point** | L4/L5에 놓으면 안정! 하지만 L1~L3는 불안정하여 조금만 밀어도 도망간다 |
| **XR 증폭** | 회전 좌표계에서 중력 포텐셜을 3D 지형으로 시각화 — 안장점(saddle point)에 서본다 |
| **Discover 핵심** | 라그랑주 점, 3체 제한 문제, 유효 포텐셜, 안정성(L4/L5 트로이 소행성) |
| **난이도** | 4 |

### #5 삼체 카오스 (Three-Body Chaos)

| 항목 | 내용 |
|------|------|
| **컨셉** | 질량이 같은 3개의 별을 배치하고 초기 속도를 줘서 운동을 관찰 |
| **Must 3D** | 3체 문제의 궤적은 3D. 8자형 궤도 같은 특수해는 3D에서만 시각화 가능 |
| **변수 세팅** | `bodies=3`, `mass=[1,1,1]`, `initialPositions=variable`, `initialVelocities=variable` |
| **예측 (trajectory)** | "이 세 별은 1000년 후에 어디에 있을까? 궤적을 그려봐" |
| **God Hand** | 별 하나의 초기 위치를 아주 살짝 바꿔본다 — 결과가 완전히 달라진다 |
| **WOW Point** | 초기 조건을 0.001%만 바꿔도 결과가 완전히 다르다 — 카오스! "이래서 삼체 문제가 풀리지 않는다" |
| **XR 증폭** | 3개의 빛나는 별 사이에 서서 빠르게 지나가는 별을 피하며 관찰 |
| **Discover 핵심** | 삼체 문제, 카오스 이론, 초기 조건 민감성, 특수해(8자형 궤도) |
| **난이도** | 5 |

### #6 쌍성 댄스 (Binary Star Dance)

| 항목 | 내용 |
|------|------|
| **컨셉** | 두 별의 질량과 거리를 조절하여 쌍성계를 만들고, 공통 질량 중심을 찾는다 |
| **Must 3D** | 쌍성의 궤도면은 3D 기울기를 가짐. 시선 방향에 따라 식쌍성/분광쌍성이 됨 |
| **변수 세팅** | `star1={mass:2e30}`, `star2={mass:1e30}`, `distance=variable`, `orbitPlane=3D` |
| **예측 (placement)** | "이 두 별의 공통 질량 중심은 어디? 거기에 마커를 놓아봐" |
| **God Hand** | 별의 질량을 손으로 키우면 질량 중심이 이동하는 것이 보인다. 궤도면 기울기 조절 |
| **WOW Point** | 한 별의 질량을 엄청 키우면 — 작은 별이 거대한 별 표면 근처를 맴돌며 질량을 빨아들인다 (질량 전달) |
| **XR 증폭** | 공통 질량 중심에 서서 두 별이 자기 주위를 도는 것을 관찰. 가스 스트림 시각화 |
| **Discover 핵심** | 쌍성계, 질량 중심, 질량 전달, 로슈엽(Roche lobe), 식쌍성 |
| **난이도** | 3 |

### #7 은하 충돌 (Galaxy Collision)

| 항목 | 내용 |
|------|------|
| **컨셉** | 두 은하의 접근 각도와 속도를 설정하여 병합 과정을 시뮬레이션 |
| **Must 3D** | 은하 충돌의 결과(안테나 은하 등)는 3D 접근 각도에 의존. 정면/측면/비스듬한 충돌 모두 다른 결과 |
| **변수 세팅** | `galaxy1={stars:100000,type:spiral}`, `galaxy2={stars:50000,type:spiral}`, `approachAngle=3Dvec`, `relativeVelocity=vec3` |
| **예측 (pattern)** | "이 각도로 충돌하면 결과 은하는 어떤 모양이 될까?" |
| **God Hand** | 은하를 손으로 잡고 던져서 충돌시킨다. 접근 각도와 속도가 결과를 결정 |
| **WOW Point** | 수십만 개의 별이 충돌하는데 — 실제로 별끼리 부딪칠 확률은 거의 0! 별 사이 공간이 그만큼 넓다 |
| **XR 증폭** | 은하 중심에 서서 수만 개의 별이 지나가는 것을 목격. 조석 꼬리(tidal tail) 형성 과정 |
| **Discover 핵심** | 은하 병합, 조석 상호작용, 별 사이 평균 거리, 동적 마찰 |
| **난이도** | 4 |

### #8 스케일 계단 (Scale Staircase)

| 항목 | 내용 |
|------|------|
| **컨셉** | 원자 → 세포 → 인간 → 지구 → 태양계 → 은하 → 관측 가능 우주까지 확대하며 각 규모의 현상을 체험 |
| **Must 3D** | 우주의 규모감은 3D 공간에서만 체감 가능. 지수적 스케일 변화 |
| **변수 세팅** | `startScale=1e-10`, `endScale=8.8e26`, `steps=10`, `currentScale=variable` |
| **예측 (pattern)** | "이 규모에서 10배 확대하면 무엇이 보일까?" |
| **God Hand** | 핀치 제스처로 줌 인/아웃. 각 스케일에서 대표 물체가 나타난다 |
| **WOW Point** | 원자에서 관측 가능 우주까지 10^36배 차이. 매 스케일마다 전혀 다른 세계 |
| **XR 증폭** | 자기 몸 크기와 비교하며 스케일을 느낀다. 목성 옆에 서면 자신이 점보다 작다 |
| **Discover 핵심** | 우주의 계층 구조, 지수적 스케일, 각 규모의 지배적 힘(핵력/전자기력/중력) |
| **난이도** | 1 |
